### [1] [A detailed algorithmic study on a reuse-aware, near memory, all-digital Ising machine](https://arxiv.org/abs/2605.12959)
*Siddhartha Raman Sundara Raman,Lizy K. John,Jaydeep P. Kulkarni*

Main category: cs.AR

TL;DR: 本文提出SACHI，一种通过重用CPU的L1缓存实现的全数字Ising加速架构，利用SRAM存内计算技术，相比BRIM实现了300倍性能提升和80倍能效降低，并支持多种NP-complete优化问题。


<details>
  <summary>Details</summary>
Motivation: 现有Ising加速器要么局限于特定问题类型，要么依赖ADC/DAC电路，且受工艺波动影响的嵌入式存储技术存在可靠性挑战。作者希望通过全数字、紧耦合的方式解决这些问题。

Method: SACHI通过重用CPU的L1缓存实现SRAM存内计算，采用全数字设计（无需ADC/DAC），紧密集成于CPU流水线中，并提出重用感知的计算和映射策略以提升并行度。

Result: 相比BRIM，SACHI实现300倍性能提升和80倍能效降低；在资产分配、分子动力学、图像分割和旅行商等应用中观察到最高4000倍的重用因子。

Conclusion: 通过紧耦合通用处理器的商用SRAM结构，可以实现可靠且高效的全数字Ising加速，为大规模高精度优化问题提供实用方案。

Abstract: Recently, nature-inspired computing approaches have gained significant attention for solving difficult optimization problems, particularly through Ising machines for NP-complete applications. Existing Ising accelerators range from quantum and optical annealers to CMOS-based von-Neumann and in-memory architectures. However, many prior designs are specialized accelerators limited to specific problem classes, rely on ADC/DAC circuits, and suffer from reliability challenges due to process-variation-sensitive embedded memory technologies.
  This paper presents SACHI, an all-digital Ising architecture implemented by repurposing the L1 cache of a CPU using SRAM-based processing-in-memory techniques. SACHI eliminates the need for ADCs/DACs, improves reliability compared to prior approaches such as BRIM, and enables Ising acceleration with minimal hardware overhead integrated into the CPU pipeline. The paper also provides detailed architectural analysis and pseudo-code for the proposed algorithms.
  The key contributions of SACHI are: (i) tight integration of the accelerator with the CPU pipeline, (ii) reuse of existing cache hardware for acceleration, (iii) higher parallelism enabled through reuse-aware computation, and (iv) improved performance and energy efficiency for large-scale, high-precision optimization problems using novel compute and mapping strategies.
  Compared to BRIM, SACHI achieves 300x performance improvement and 80x energy reduction across applications including asset allocation, molecular dynamics, image segmentation, and traveling salesman problems. Additionally, reuse factors up to 4000x are observed for several workloads. This work demonstrates that reliable and efficient all-digital Ising acceleration can be achieved using commodity SRAM structures tightly integrated with general-purpose processors.

### [2] [PoisonCap: Efficient Hierarchical Temporal Safety for CHERI](https://arxiv.org/abs/2605.13210)
*Yuecheng Wang,Jonathan Woodruff,Alfredo Mazzinghi,Peter Rugg,Alexandre Joannou,Samuel W. Stark,Robert N. M. Watson,Simon W. Moore*

Main category: cs.AR

TL;DR: PoisonCap 是一种针对 CHERI 系统的可扩展时间安全方案，通过新型"毒化"能力格式实现严格的 use-after-free 保护和初始化安全，在不增加性能开销的前提下增强了 CHERI 的时间安全性。


<details>
  <summary>Details</summary>
Motivation: 现有 CHERI 时间安全方案（Cornucopia Reloaded）仅提供 use-after-reallocation 保护而非更强的 use-after-free 保护，且无法强制执行初始化安全。需要一种更严格的时间安全方案。

Method: 提出新型"毒化"能力（poison capability）格式，利用能力边界实现内存毒化权限的优雅委托，支持嵌套分配器在不干扰上游分配器的情况下对消费者实施安全策略。通过毒化能力向微架构通信内存状态，实现隔离内存的高效缓存管理。

Result: PoisonCap 可替代 Cornucopia 的影子位图，在重新分配时自动清零内存，或可选择在读前写时触发陷阱以强制执行初始化安全。与在重新分配前清零的 Cornucopia 基线相比，无根本性性能开销。

Conclusion: PoisonCap 在不引入性能开销的前提下，为 CHERI 系统提供了更强的 use-after-free 保护和初始化安全，是对现有时间安全方案的重要改进。

Abstract: In this paper, we present PoisonCap: scalable temporal safety with strict use-after-free protection and initialisation safety for CHERI systems. Efficient memory safety is an increasing priority for programming languages, operating systems, and hardware designs, and CHERI is a leading hardware/software system that provides native spatial safety and a foundation for temporal memory safety. Cornucopia Reloaded, the current state-of-the-art CHERI temporal safety solution, provides use-after-reallocation safety instead of stronger use-after-free safety, and is not able to enforce initialisation safety. We show that a new 'poison' capability format can be used to enforce strict use-after-free and initialisation safety, and also to communicate memory state to the microarchitecture for efficient cache management of quarantined memory. We enable elegant delegation of memory poisoning privilege using capability bounds to allow nested allocators to enforce safety on their consumers without disturbing upstream allocators. PoisonCap can replace the Cornucopia shadow bitmap, and also automatically zeros memory on reallocation, or optionally traps on read-before-write to enforce initialisation safety. As a result, it incurs no fundamental overhead relative to a Cornucopia baseline that zeros before reallocation, strengthening CHERI temporal safety without performance overhead.

<details>
  <summary>Details</summary>
Motivation: CPU-based OLTP存在锁服务效率低下的问题，因为大多数锁是冷锁，锁代理需要频繁内存访问来获取锁细节以决定是否授予锁。

Method: 1) 提出低延迟的硬件锁代理，优化锁获取和释放请求；2) 设计可扩展的事务代理，执行完整的事务生命周期；3) 提出架构、优化和设计空间探索方案。

Result: 在TPC-C基准测试上，相比CPU基线实现最高51倍的事务吞吐量提升。

Conclusion: 专用硬件加速器通过集成锁表消除DRAM访问开销，能显著提升OLTP系统的锁管理效率和事务处理性能。

Abstract: Online Transaction Processing (OLTP) is a classic application with a growing business. CPU-based OLTP has low lock serving efficiency. This motivates us to propose dedicated hardware-based lock agents with integrated lock tables to remove the DRAM access overhead.
  In this paper, we propose hardware-accelerated lock management and transaction processing for database systems. First, we propose a low-latency lock agent optimized for both lock acquiring and releasing requests. Second, we design a scalable transaction agent that executes the full transaction lifecycle. We present the architecture, optimizations, and design-space exploration of the proposed lock management and transaction processing system. The experiment results show up to 51X higher transaction throughput over the CPU baseline on the TPC-C benchmark.

### [4] [Reward-Weighted On-Policy Distillation with an Open Property-Equivalence Verifier for NL-to-SVA Generation](https://arxiv.org/abs/2605.13501)
*Qingyun Zou,Yingze Li,Tianen Liu,Bingsheng He,Weng-Fai Wong*

Main category: cs.AR

TL;DR: 本文指出现有LLM生成SystemVerilog断言（SVA）的评估掩盖了时序漏洞，提出Reward-Weighted On-Policy Distillation（RWOPD）方法，利用属性等价检查器作为奖励信号进行知识蒸馏，蒸馏后的7B模型在多个指标上超越专用大模型和671B通用大模型，达到新SOTA。


<details>
  <summary>Details</summary>
Motivation: 现有LLM生成SVA的评估指标（如76%准确率）看似饱和，但掩盖了模型在有界延迟和活性规范上仅能生成少数固定蕴含模板的问题。根本原因在于监督微调优化的是token级模仿，而非真正的属性等价性（property equivalence），导致模型缺乏对SVA正确性的深层理解。

Method: 提出Reward-Weighted On-Policy Distillation（RWOPD），一种在线策略蒸馏方法：1）学生模型进行rollout采样生成SVA；2）使用开源的SymbiYosys+Z3属性等价检查器（PEC）对rollout评分；3）对验证器通过的rollout，使用冻结的14B教师模型施加基于验证器奖励加权的forward-KL梯度，在每一个响应token上提供密集监督。

Result: 将CodeV-SVA-14B蒸馏到Qwen2.5-Coder-7B-Instruct后，在NL2SVA-Human和NL2SVA-Machine上，pass@1、pass@5和pass@10指标均达到新SOTA，超越此前专用SOTA模型和671B通用大模型。

Conclusion: 通过引入属性等价检查器作为奖励信号，结合在线策略蒸馏，可以有效解决LLM生成SVA中"看似饱和实则欠拟合"的问题，验证了用形式化验证而非token级模仿来指导模型训练的有效性。

Abstract: LLM-based generation of SystemVerilog Assertions (SVA) is often reported as nearing saturation, with the strongest specialized model reaching ${\sim}76\%$ accuracy on NL2SVA-Human. We show that this aggregate hides a temporal gap: models that appear strong overall still collapse to a few implication templates on bounded-delay and liveness specifications. The core issue is that the dominant recipe, supervised fine-tuning on NL/SVA pairs, optimizes token-level mimicry rather than the \emph{property equivalence} that defines SVA correctness. We introduce \emph{Reward-Weighted On-Policy Distillation} (RWOPD), an on-policy distillation method that samples student rollouts, scores them with an open SymbiYosys+Z3 Property-Equivalence Checker (PEC), and applies a verifier-reward-weighted forward-KL gradient from a frozen 14B teacher on verifier-passable rollouts. This keeps the supervision dense at every response token while grounding both selection and loss weight in property-equivalent behavior. RWOPD distills CodeV-SVA-14B into a Qwen2.5-Coder-7B-Instruct student that sets a new state of the art on NL2SVA-Human and NL2SVA-Machine across pass@1, pass@5, and pass@10, surpassing both specialized prior SOTA models and 671B general-purpose baselines.

### [5] [Efficient Implementation of an Adaptive Transformer Accelerator for Massive MIMO Outdoor Localization](https://arxiv.org/abs/2605.13507)
*Ilayda Yaman,Sijia Cheng,Ove Edfors,Liang Liu*

Main category: cs.AR

TL;DR: 本文提出了一种面向5G大规模MIMO的自适应Transformer定位系统硬件实现，利用波束时延信道表示的稀疏性实现行跳跃机制，结合混合数据流架构和轻量级运行时模型切换，在FPGA上实现了亚毫秒级实时定位。


<details>
  <summary>Details</summary>
Motivation: 5G大规模MIMO场景下，基于Transformer的定位模型计算量大、延迟高，难以满足实时定位（sub-10ms）的需求，因此需要高效的硬件加速方案。

Method: 1) 利用传播信道稀疏性，设计行跳跃（row-wise skipping）机制去除低能量波束分量；2) 混合数据流架构（输入驻留+输出驻留）映射到异构向量处理引擎；3) 轻量级运行时模型切换机制，使用单层感知机路由器的时域滤波输出实现模型稳定选择。

Conclusion: 传播感知稀疏性、混合数据流执行和高效运行时模型切换可实现面向5G实时定位的自适应Transformer可扩展低延迟硬件实现。

Abstract: We present the implementation of an adaptive Transformer-based localization system for 5G massive MIMO targeting sub-10ms real-time positioning. The design exploits propagation characteristics, where beam-delay channel representations exhibit sparsity, enabling a row-wise skipping mechanism that removes low-energy beam components with minimal control overhead. The contribution is focused on hardware realization of the model using a mixed dataflow architecture, combining input- and output-stationary execution, mapped onto a heterogeneous vector processing engine with parallel processing elements and adder trees for efficient matrix computation. Environment-dependent processing is supported through a lightweight runtime model-switching mechanism, where temporally filtered outputs of a single-layer perceptron router enable stable selection between specialized models with reduced latency. Implemented on a Xilinx Zynq UltraScale+ FPGA and evaluated on real-world massive MIMO measurements, the design achieves up to 65% row sparsity, yielding peak computational speedups of approximately 2x while limiting the average localization accuracy degradation to below 10%, relative to the floating-point baseline model. The accelerator attains below 1.15m localization accuracy across scenarios, with inference latency of 0.51-2.11ms and throughput of up to 1961 positions/s. These results demonstrate that propagation-aware sparsity, mixed dataflow execution, and efficient runtime model switching enable a scalable and low-latency hardware realization of adaptive Transformer-based localization for real-time 5G systems.
