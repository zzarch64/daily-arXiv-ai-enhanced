### [1] [Design and Implementation of BNN-Based Object Detection on FPGA](https://arxiv.org/abs/2605.03396)
*Xuyu Zhao,Yunpeng Wu,Mengyuan Zhu,Haoyu Huang,Xiaoyu Xu,Yanjing Li,Gaolong Zhang,Baochang Zhang*

Main category: cs.AR

TL;DR: 本文在低成本FPGA上实现了基于二值神经网络(BNN)的YOLOv3-tiny目标检测器，采用1-bit权值和8-bit激活值，通过Verilog RTL硬件设计，在VOC数据集上达到39.6% mAP50，硬件仿真与ONNX模型相关性达0.999964。


<details>
  <summary>Details</summary>
Motivation: 为了在资源受限的低成本FPGA上实现高效的目标检测，通过二值神经网络大幅降低模型存储和计算开销，同时保持可接受的检测精度。

Method: 采用BNN YOLOv3-tiny结构，主卷积层使用1-bit权重和8-bit激活值；将ONNX模型参数转为定点数并存入BRAM ROM；用Verilog RTL实现padding、行缓冲、二值卷积、量化后处理、最大池化和检测头计算；将Mul_prev融合到BNN PE中实现逐通道补偿。

Result: VOC数据集上mAP50为39.6%，模型仅0.098 GFLOPs和0.74M参数；RTL仿真与ONNX节点输出相关系数达0.999964，平均绝对误差0.020027。

Conclusion: 该设计证明了在低成本FPGA上部署BNN目标检测器的可行性，并在硬件精度和模型效率之间取得了良好的平衡。

Abstract: This paper implements a Binary Neural Network (BNN) based YOLOv3-tiny-like object detector on a low-cost FPGA. The network takes 320*320*3 RGB images as input. Its main convolution layers use 1-bit weights and 8-bit activations, while Conv1 and the final detection head use fixed-point standard convolutions. From the trained ONNX model, weights, biases, and quantization parameters are extracted, converted to fixed point, packed into COE files, and stored in Vivado BRAM ROMs. The hardware is written fully in Verilog RTL and includes padding, line buffering, binary convolution, quantization post-processing, max pooling, and detection-head computation. For layers where Mul_prev is indexed by input channel and Div_current by output channel, Mul_prev is fused in-to the BNN PE so that channel-wise compensation is applied during accumulation. On VOC, the model obtains 39.6% mAP50 with 0.098 GFLOPs and 0.74 M parameters. RTL simulation shows that the final raw detection output reaches a correlation coefficient of 0.999964 and a mean absolute error of 0.020027 against the corresponding ONNX node.

### [2] [SPEC CPU2026: Characterization, Representativeness, and Cross-Suite Comparison](https://arxiv.org/abs/2605.03713)
*Ruihao Li,Andrew Jacob,Neeraja J. Yadwadkar,Lizy K. John*

Main category: cs.AR

TL;DR: 本文首次全面分析了SPEC CPU2026基准测试套件，在九种现代处理器平台上与SPEC CPU2017、DCPerf、MLPerf进行对比。研究发现SPEC CPU2026增加了指令量和内存占用，加剧了指令缓存压力；通过聚类分析可用4-5个负载子集保留96.4-99.9%的套件行为；相比MLPerf向量化程度更低、比DCPerf前端压力更小，更接近真实CPU行为；并通过页面大小、预取、编译优化等案例研究展示了其实用价值。


<details>
  <summary>Details</summary>
Motivation: CPU性能日益成为端到端系统效率的关键瓶颈，但现有CPU基准测试套件是否反映现代工作负载和瓶颈尚不明确。本文旨在填补SPEC CPU2026在多个现代平台上的系统性表征空白，并回答是否需要完整套件进行架构评估。

Method: 在Intel、AMD、Ampere、Nvidia九种处理器上进行全面实验；使用聚类分析进行代表性评估，识别紧凑子集；通过跨套件微架构指标对比SPEC CPU2026与CPU2017、DCPerf、MLPerf；通过页面大小、分配器、预取、编译优化、ISA敏感性、多核扩展等案例研究验证实用性。

Result: SPEC CPU2026相比CPU2017指令量增大、内存占用增加、指令缓存压力加重；4-5个负载子集可保留96.4-99.9%的全套行为；SPEC CPU2026向量化程度低于MLPerf，前端压力低于DCPerf；轮询交替模式可生成逼近DCPerf的代理负载，IPC差距缩小至13.7%。

Conclusion: SPEC CPU2026比前代更接近真实CPU行为，为严格且经济高效的CPU评估奠定了新基础。紧凑子集方案可在不牺牲保真度前提下大幅降低评估成本。

Abstract: Specialized accelerators dominate AI workloads, but CPUs remain critical for orchestrating these accelerators and running datacenter services. As a result, CPU performance increasingly shapes end-to-end system efficiency, making it necessary for benchmarks to reflect modern workloads and bottlenecks. However, it remains unclear how emerging CPU benchmark suites reflect these shifts. To address this, we present the first comprehensive characterization of SPEC CPU2026 across nine platforms spanning recent Intel, AMD, Ampere, and Nvidia processors. We find that, compared to SPEC CPU2017, SPEC CPU2026 increases instruction volume and memory footprint, and shifts pressure toward emerging bottlenecks, most notably higher instruction-cache stress. We next examine whether the full suite is necessary for architectural evaluation. Using clustering-based representativeness analysis, we identify that compact subsets of 4-5 workloads per group preserve 96.4-99.9% of full-suite behavior, substantially reducing evaluation costs without sacrificing fidelity. To better position SPEC CPU2026, we compare it against SPEC CPU2017, DCPerf, and MLPerf using cross-suite microarchitectural metrics. SPEC CPU2026 remains a general-purpose suite with complementary characteristics: it is less vector-intensive than MLPerf and has lower frontend pressure than DCPerf, yet moves closer to real-world CPU behavior than prior SPEC CPU generations. Finally, we show that SPEC CPU2026 supports practical architectural studies beyond aggregate scores through case studies on page sizes and allocators, prefetching, compiler optimizations, ISA sensitivity, and many-core scaling. The new round-robin stagger mode generates proxy workloads that approximate DCPerf, reducing the IPC gap to 13.7%. Overall, SPEC CPU2026 sets a new foundation for rigorous and cost-effective CPU evaluation.
