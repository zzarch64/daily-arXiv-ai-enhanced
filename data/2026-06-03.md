### [1] [Heterogeneous Mapping for Analog In-Memory Computing Accelerators: A Unified Workflow](https://arxiv.org/abs/2606.02672)
*Corey Lammie*

Main category: cs.AR

TL;DR: 本文分类了将DNN工作负载分区到模拟存算一体（AIMC）与数字处理单元的异构架构方法，提炼出统一的四阶段工作流，并首次针对GPT-2生成AIMC精度灵敏度分析，发现49个投影中仅4个主导灵敏度，首个解码器块的注意力输出尤其关键。


<details>
  <summary>Details</summary>
Motivation: AIMC加速器可减少数据搬移、提升DNN推理效率，但其有效精度有限，因此需要异构架构（模拟+数字）来平衡效率与精度。现有分区方法多样但缺乏统一框架，且尚无针对解码器类Transformer（如GPT-2）的AIMC精度敏感性研究。

Method: 首先对现有AIMC-DNN分区方法按映射粒度、优化策略和模型支持进行分类，并提炼为四阶段工作流；然后将其前两阶段应用于GPT-2，生成首个面向解码器Transformer的AIMC精度灵敏度分析。

Result: GPT-2的49个投影中，仅有4个主导精度灵敏度；其中第一个解码器块的注意力输出的灵敏度比其他高一个数量级。这表明投影级映射和对早期块及输出相关投影的选择性数字执行对可靠部署至关重要。

Conclusion: 投影级映射策略与选择性数字执行是AIMC上可靠部署解码器Transformer的关键方向，尤其需优先处理早期块和面向输出的投影。

Abstract: Analog In-Memory Computing (AIMC) accelerators execute matrix-vector multiplications directly within memory arrays, reducing data movement and improving DNN inference efficiency. Their limited effective precision motivates heterogeneous architectures that combine analog compute tiles with digital processing units. This letter classifies existing methods for partitioning DNN workloads across these resources by mapping granularity, optimization strategy, and model support, and distills them into a unified four-stage workflow. To demonstrate the workflow on a model class not yet addressed by existing methods, we apply its first two stages to GPT-2, producing the first AIMC-specific precision sensitivity profile for a decoder-only transformer. Sensitivity is dominated by 4 of 49 projections, with the first decoder block's attention output dominating by an order of magnitude. This suggests that projection-level mapping and selective digital execution of early-block and output-facing projections are important for reliable decoder-transformer deployment on AIMC hardware.

### [2] [CRAM-ER: Error-Resilient Spintronic Computational Random Access Memory for Scalable In-Memory Computation](https://arxiv.org/abs/2606.02781)
*Sohan Salahuddin Mugdho,Md. Shahedul Hasan,Brahmdutta Dixit,Yang Lv,Jian-Ping Wang,Cheng Wang*

Main category: cs.AR

TL;DR: 本文提出了一种基于MRAM的容错CRAM架构（CRAM-ER），通过混合自旋-CRAM+CMOS加法树设计及误差感知的软硬件协同优化，在实现高效存内矩阵向量乘法的同时，将延迟降低两个数量级，并在DNN基准测试中保持近乎无损的精度，能效和能量延迟积均优于CPU/GPU+高带宽DRAM方案。


<details>
  <summary>Details</summary>
Motivation: 传统冯·诺依曼架构面临严重的内存瓶颈，现有近内存和存内计算方案外围开销大。基于MRAM的计算随机存取存储器（CRAM）可实现原位逻辑且无外围开销，但存在两个核心问题：概率性MRAM开关导致的逻辑门级错误限制了DNN加速的可扩展性和可靠性；大量顺序MRAM写入严重制约了CRAM的吞吐量。

Method: 提出CRAM-ER架构，核心包括：1）混合自旋-CRAM+CMOS加法树混合架构，在减轻器件级错误影响的同时保持高面积和能效；2）误差感知的硬件-软件协同设计框架；3）误差感知的模型微调和细粒度纠错机制，增强误差鲁棒性。

Result: 在DNN基准测试中，CRAM-ER实现了近乎无损的精度，同时将CRAM延迟降低了最多两个数量级；在能效和能量延迟积方面均优于CPU/GPU+高带宽DRAM方案。

Conclusion: 通过硬件-软件协同的误差感知设计，成功解决了CRAM在DNN加速中的可靠性和吞吐量瓶颈，为存内计算在深度学习领域的大规模应用提供了可行方案。

Abstract: Deep neural networks (DNNs) have achieved state-of-the-art performance across diverse domains. However, typical Von Neumann compute paradigms face severe memory bottlenecks. Emerging near-memory and compute-in-memory approaches alleviate this but incur significant peripheral overhead. Computational Random Access Memory (CRAM) based on MRAM enables in-situ logic without peripheral overhead, offering a dense, energy-efficient solution. However, probabilistic MRAM switching induces gate-level errors that limit the scalability and reliability of CRAM for accelerating DNN. Moreover, the large number of sequential MRAM writes severely constrains CRAM throughput. To address these challenges, we propose an error-resilient CRAM (CRAM-ER) architecture for scalable in-memory matrix-vector multiplications (MVMs). Our error-aware hardware-software co-design framework leverages a hybrid spintronic-CRAM + CMOS adder-tree architecture to mitigate the impact of device-level errors, demonstrating MVM functionality with high area and energy efficiency. We further develop an error-aware model fine-tuning and fine-grained error correction for enhanced error resilience. Evaluations of the CMOS+spintronic hybrid architecture on DNN benchmarks show near-lossless accuracy while reducing CRAM latency by up to 2 orders of magnitude, outperforming CPU/GPU+high-bandwidth DRAM in both energy efficiency and energy-delay product.

<details>
  <summary>Details</summary>
Motivation: Transformer模型在资源受限的边缘设备上部署困难，现有ARM-CL框架只支持CNN推理，缺乏对Transformer推理的原生支持。

Method: 1) 在ARM-CL中实现新的Transformer核；2) 提出协同CPU-GPU推理策略，将内存密集型操作分配给CPU，计算密集型并行操作分配给GPU。

Result: 扩展的ARM-CL比现有CPU/GPU实现快3倍；协同推理相比最佳单处理器方案再降低15.72%延迟。

Abstract: Transformer models have set new performance standards for machine learning (ML) tasks. However, their resource-intensive deployment on resource-constrained edge devices for cloud-free, on-chip transformer inference remains challenging. The ARM Compute Library (ARM-CL) framework provides low-latency CNN inference on ARM-based edge devices but lacks support for transformer inference. In this work, we implement several new transformer kernels in ARM-CL to support native transformer execution. Our extended ARM-CL achieves up to three times faster transformer inference compared to state-of-the-art CPU/GPU implementations on an ARM-based embedded board.
  Furthermore, heterogeneous multi-processor system-on-chips (HMPSoCs) powering edge devices provide both embedded CPUs and GPUs. We introduce cooperative CPU-GPU transformer inference, which executes memory-intensive operations on the CPU while utilizing the GPU for highly parallelizable, compute-intensive operations. This cooperative execution, implemented with minimal overhead, further reduces transformer inference latency by up to 15.72% compared to the best single-processor inference on ARM-CL.

### [4] [Multi-Segment Attention: Enabling Efficient KV-Cache Management for Faster Large Language Model Serving](https://arxiv.org/abs/2606.02964)
*Chunan Shi,Yilei Chen,Yilin Chen,Xupeng Miao,Bin Cui*

Main category: cs.AR

TL;DR: 提出AsymCache，一个计算延迟感知的KV缓存管理系统，通过多段注意力、位置感知驱逐策略和自适应分块调度，在保持无损精度的同时显著提升LLM推理效率。


<details>
  <summary>Details</summary>
Motivation: 现有无损KV缓存管理主要基于访问频率或位置启发式进行驱逐决策，未考虑不同KV缓存块对GPU注意力核执行效率的影响，导致计算效率和缓存命中率无法兼顾。

Method: 1) 多段注意力(MSA)：高效处理非连续KV上下文；2) 联合优化命中率和位置感知重计算成本的缓存驱逐策略；3) 自适应分块调度器以提高硬件利用率。

Conclusion: 计算延迟感知的缓存管理能有效平衡计算效率与缓存命中率，且低层设计可无缝集成到现有代理服务系统。

Abstract: Large Language Model (LLM) inference relies on key-value (KV) caches to avoid redundant attention computation. While approximate KV cache retention techniques reduce memory usage by sacrificing model accuracy, lossless approaches instead evict KV cache blocks from GPU memory and reconstruct them on demand to preserve exact outputs. Existing lossless KV cache management systems primarily base eviction decisions on access frequency or positional heuristics, without considering how different KV cache blocks affect the execution efficiency of GPU attention kernels.
  In this paper, we propose AsymCache, a computation-latency-aware KV cache management system for LLM inference that explicitly aligns cache residency decisions with GPU attention kernel performance, including three key components: Multi-Segment Attention (MSA) for efficient non-contiguous KV context processing, a cache eviction policy that jointly optimizes hit rate and position-aware recomputation cost, and an adaptive chunking scheduler for high hardware utilization. Experiments show that AsymCache reduces TTFT by up to 1.90-2.03x and time-per-output-token (TPOT) by 1.62-1.71x over latest baselines, confirming the effectiveness of the method in common workloads and validating its design goal of balancing computational efficiency with cache hit rate. Moreover, the low-level design of AsymCache allows seamless integration into agent serving systems such as Continuum, where it further reduces average job latency by up to 18.1%.

### [5] [ZK-Flex: A Flexible and Scalable Framework for Accelerating Zero-Knowledge Proofs](https://arxiv.org/abs/2606.03046)
*Adiwena Putra,Cuong Manh Duong,Anh Quang Pham,Joo-Young Kim*

Main category: cs.AR

TL;DR: ZK-Flex是一个软硬件协同设计的框架，通过TCore多精度计算核心、灵活NoC和链表内存机制，在ZKP证明生成中实现5-11倍加速和最高3.8倍面积效率提升。


<details>
  <summary>Details</summary>
Motivation: ZKP证明生成计算密集，主要由多项式(POLY)和椭圆曲线(EC)运算构成，现有可重构加速器在精度可扩展性、算法灵活性和资源效率方面存在局限。

Method: 提出ZK-Flex框架：软件层包含POLY和EC优化器，通过硬件感知和负载感知的算法选择减少计算量；硬件层集成基于Toom-Cook的多精度核心TCore，配备灵活NoC和链表内存机制以提升有限内存下的并行度。

Abstract: Zero-knowledge proofs (ZKP) allows a prover to convince a verifier of computational correctness without revealing private data, ensuring both privacy and verifiability. However, proof generation is highly compute-intensive, dominated by polynomial (POLY) and elliptic-curve (EC) operations. These workloads pose two key challenges for hardware acceleration: (1) efficiently supporting diverse large-precision modular multiplications, and (2) maintaining high utilization across workloads that dynamically shift between POLY and EC stages. Existing reconfigurable accelerators address these issues only partially, remaining limited in precision scalability, algorithmic flexibility, and resource efficiency.
  To overcome these limitations, we propose ZK-Flex, a flexible and scalable software-hardware co-designed framework for accelerating ZKP proof generation. The software layer incorporates POLY and EC optimizers that reduce computation through hardware- and workload-aware algorithmic choices, while the hardware integrates TCore, a Toom-Cook-based multi-precision core with a flexible NoC and a linked-list memory mechanism that improves parallelism under limited memory capacity. Across representative ZKP benchmarks, ZK-Flex achieves 5 to 11 times speedup and up to 3.8 times higher area efficiency over the state of the art, establishing a new foundation for high-performance, reconfigurable ZKP acceleration.

### [6] [ACRONYM: Accelerated Approximate Nearest Neighbor Search in Memory for Dynamic Vector Databases](https://arxiv.org/abs/2606.03151)
*Md Mizanur Rahaman Nayan,Tianqi Zhang,Flavio Ponzina,Tajana Rosing,Azad J Naeemi*

Main category: cs.AR

TL;DR: 提出了一种算法-硬件协同设计的平台ACRONYM，通过数据分布无关的编码、CAM内存并行距离计算、两级搜索策略和XAC脉动阵列编码器，在支持动态更新的向量数据库中实现了>90%召回率、8e6 QPS吞吐量，相比HNSW(CPU)加速约400倍、FAISS-IVF(GPU)加速约80倍。


<details>
  <summary>Details</summary>
Motivation: 现有图基和分区基的近似最近邻搜索方案因依赖数据分布而导致频繁索引重建，连续部署困难且重建延迟长，无法高效支持频繁更新的向量数据库搜索。

Method: 1) 数据分布无关的高效编码 + 基于汉明距离的硬件加速搜索；2) CAM基的内存并行距离计算 + 时域复用的近似top-k选择实现穷举搜索；3) 两级搜索策略（粗搜索+二值精化）解决CAM容量限制；4) 新颖的XOR-Accumulate（XAC）脉动阵列编码器实现片上高效编码。

Result: 百万级数据集上，动态数据库场景下达到>90%召回率、8e6 QPS吞吐量、32MB内存占用、2.56uJ/查询平均能耗，相比HNSW(CPU)加速约400倍，相比FAISS-IVF(GPU)加速约80倍。

Conclusion: ACRONYM通过算法-硬件协同设计，在支持无中断连续更新的同时实现了高性能、低能耗的向量数据库搜索，显著优于现有CPU和GPU方案。

Abstract: Vector database search with frequent updates is increasingly critical in applications such as retrieval augmented generation, recommendation systems, and large-scale embedding retrieval. Existing solutions, such as graph-based and partition-based approximate nearest neighbor search (ANNS), suffer from frequent index rebuilding due to data distribution-dependent indexing that impacts continuous deployment and causes long rebuilding latency. This paper proposes an algorithm-hardware co-designed platform, ACRONYM, that addresses key problems with state of the art database search. Algorithmically, it leverages efficient encoding independent of data distribution and Hamming-distance based search for efficient hardware acceleration. Architecturally, we propose CAM-based in-memory parallel distance computation followed by time multiplexed approximated top-k selection to enable the exhaustive search. We propose two-stage search that includes coarse search followed by binary refinement to achieve high recall in CAM based search which is heavily limited to small vector dimension due to capacity and wordline parasitic. ACRONYM supports continuous update without stalling and integrates novel XOR-and-Accumulate (XAC) based systolic-array encoder for efficient on chip encoding during search. Across million-scale datasets, while serving dynamic database ACRONYM achieves >90% recall at a throughput of 8e6 queries per second, with a memory footprint of only 32MB and an average energy consumption of 2.56uJ per query, speedup over HNSW (CPU) of about 400x and FAISS-IVF (GPU) of about 80x.
