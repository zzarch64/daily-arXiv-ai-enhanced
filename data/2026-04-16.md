### [1] [Tensor Memory Engine: On-the-fly Data Reorganization for Ideal Locality](https://arxiv.org/abs/2604.13319)
*Denis Hoornaert,Cole Strickler,Manos Athanassoulis,Marco Caccamo,Heechul Yun,Renato Mancuso*

Main category: cs.AR

TL;DR: 提出硬件/软件协同设计的Tensor Memory Engine，通过重构内存布局为CPU提供理想缓存局部性，解决数据密集型边缘计算中的内存墙问题。


<details>
  <summary>Details</summary>
Motivation: 随着计算从云端转向边缘，数据密集型处理面临内存墙挑战。大多数应用要么局部性差需要昂贵重设计，要么内存占用膨胀以维持局部性。现有系统依赖理想内存布局和缓存局部性，但实际应用难以满足这些要求。

Method: 在商用SoC/FPGA平台上实现硬件/软件协同设计，在CPU数据路径中插入Tensor Memory Engine。该引擎代表CPU访问内存并重构内存布局视图，为运行应用提供理想缓存局部性。计算仍由CPU执行，数据重组委托给Tensor Memory Engine，实现计算与内存访问的解耦。

Result: 提出的方法能够为数据密集型应用提供理想缓存局部性，无需应用重设计或内存占用膨胀。与内存内/近内存计算不同，该方法明确分离计算和内存访问，可在商用平台上实现。

Conclusion: Tensor Memory Engine通过硬件/软件协同设计有效解决边缘计算中的内存墙问题，为数据密集型应用提供理想缓存局部性，同时保持计算与内存访问的清晰分离。

Abstract: The shift to data-intensive processing from the cloud to the edge has introduced new challenges and expectations for the next generation of intelligent computing systems. As the memory wall continues to grow, modern systems can only meet these performance expectations by displaying data access patterns that exhibit ideal layouts in memory and ideal spatiotemporal locality in caches. However, only a few data-intensive applications are characterized by ideal locality. Instead, most applications exhibit either (i) poor locality when naively implemented and must undergo costly redesigns and tuning or (ii) inflated memory footprint to offer proper locality.
  To address the aforementioned challenges, we propose a hardware/software co-designed approach that can be implemented on commercially available SoC/FPGA platforms. Our approach seamlessly inserts in the CPUs' data path a Tensor Memory Engine that provides data with an ideal cache locality to running applications by (i) accessing the memory on behalf of the CPUs and (ii) composing a re-organized view of the memory layout. Unlike in- and near-memory computing approaches, it sets itself apart by clearly decoupling computing and memory accesses; computation is still performed on CPUs while the data re-organization is delegated to the Tensor Memory Engine.

### [2] [ATLAAS: Automatic Tensor-Level Abstraction of Accelerator Semantics](https://arxiv.org/abs/2604.13523)
*Ruijie Gao,Haoran Jin,Jirong Yang,Nathaniel Bleier*

Main category: cs.AR

TL;DR: ATLAAS是一个端到端的MLIR工具链，能够从RTL提取的加速器语义自动生成张量ISA规范，解决了手动编写规范的问题，实现了从硬件到编译器后端的自动化流程。


<details>
  <summary>Details</summary>
Motivation: 现有张量加速器设计大多缺乏良好文档化的ISA和编译器后端，限制了评估范围。虽然已有工作表明给定张量级ISA规范可以自动生成完整软件栈，但编写这些规范仍然是手动、专家驱动的过程。

Method: ATLAAS采用8通道语义提升流水线，从先前架构级模型提取产生的位级LLVM IR开始，逐步恢复高级张量结构（MAC习惯用法、饱和语义、多维缓冲区组织和数据布局转换），最终生成可直接用于ACT生态系统自动软件栈生成的规范。

Result: 在Gemmini脉动阵列加速器上，流水线将位级MLIR减少了92.9%（处理单元）和24-34%（控制器模块）。ATLAAS发现了手工参考规范中遗漏的硬件特性，并通过Z3 SMT等价证明验证了正确性。在TVM的VTA处理器上，同一流水线无需特定修改即可提升所有四个数据路径模块。

Conclusion: ATLAAS首次实现了从RTL到性能竞争力编译器后端的自动化路径，能够自动生成张量ISA规范，显著减少了手动工作，提高了硬件-软件协同设计的效率。

Abstract: Numerous tensor accelerator designs have been proposed, yet most lack well-documented ISAs and compiler backends, limiting evaluation to a handful of operators. Recent work has shown that given a tensor-level ISA specification, complete software stacks including compiler backends can be automatically generated--but writing such specifications remains a manual, expert-driven process.
  We present ATLAAS, the first end-to-end MLIR-based pipeline that lifts RTL-extracted accelerator semantics to tensor ISA specifications. Starting from bit-level LLVM IR produced by prior architecture-level model extraction, ATLAAS applies an 8-pass semantic lifting pipeline that progressively recovers high-level tensor structure--MAC idioms, saturation semantics, multi-dimensional buffer organizations, and data layout transformations--emitting specifications that immediately enable automatic software stack generation through the ACT ecosystem.
  We evaluate ATLAAS on the Gemmini systolic-array accelerator, where the pipeline collapses bit-level MLIR by up to 92.9% on processing elements and 24-34% on controller modules. ATLAAS discovers hardware features omitted from the hand-written reference, with correctness validated via Z3 SMT equivalence proofs. Generality is confirmed on TVM's VTA processor, where the same pipeline lifts all four datapath modules without accelerator-specific changes, enabling an automated path from RTL to a performance-competitive compiler backend.

### [3] [Cross-Layer Co-Optimized LSTM Accelerator for Real-Time Gait Analysis](https://arxiv.org/abs/2604.13543)
*Mohammad Hasan Ahmadilivani,Levent Aksoy,Mohammad Eslami,Jaan Raik,Alar Kuusik*

Main category: cs.AR

TL;DR: 提出首个针对实时步态分析的跨层协同优化LSTM加速器ASIC设计，通过软硬件协同优化在65nm工艺下实现高性能、低功耗的步态异常检测。


<details>
  <summary>Details</summary>
Motivation: LSTM神经网络在需要实时处理和边缘计算能力的医疗应用中越来越重要，步态分析检测异常步伐以防止患者跌倒是一个典型应用。现有方案在性能、功耗和面积方面难以满足严格的设计要求，需要专门的ASIC来实现高效的实时LSTM推理。

Method: 采用跨层协同优化方法：1）软件层面进行硬件感知的位宽优化以降低硬件复杂度；2）寄存器传输级探索多种设计方案；3）生成多种布局方案寻找硬件复杂度和准确性的最佳平衡。

Result: 在65nm工艺下，最高准确度优化的加速器布局面积为0.325 mm²，硬件复杂度优化的设计面积减少15.4%（准确度略低）。所有设计的加速器都能以比应用要求快4.05倍的速度实现准确的步态异常检测。

Conclusion: 成功设计了首个针对实时步态分析的跨层协同优化LSTM加速器ASIC，通过软硬件协同优化实现了高性能、小面积的步态异常检测解决方案，满足医疗应用的严格实时要求。

Abstract: Long Short-Term Memory (LSTM) neural networks have penetrated healthcare applications where real-time requirements and edge computing capabilities are essential. Gait analysis that detects abnormal steps to prevent patients from falling is a prominent problem for such applications. Given the extremely stringent design requirements in performance, power dissipation, and area, an Application-Specific Integrated Circuit (ASIC) enables an efficient real-time exploitation of LSTMs for gait analysis, achieving high accuracy. To the best of our knowledge, this work presents the first cross-layer co-optimized LSTM accelerator for real-time gait analysis, targeting an ASIC design. We conduct a comprehensive design space exploration from software down to layout design. We carry out a bit-width optimization at the software level with hardware-aware quantization to reduce the hardware complexity, explore various designs at the register-transfer level, and generate alternative layouts to find efficient realizations of the LSTM accelerator in terms of hardware complexity and accuracy. The physical synthesis results show that, using the 65 nm technology, the die size of the accelerator's layout optimized for the highest accuracy is 0.325 mm^2, while the alternative design optimized for hardware complexity with a slightly lower accuracy occupies 15.4% smaller area. Moreover, the designed accelerators achieve accurate gait abnormality detection 4.05x faster than the given application requirement.

### [4] [GEM3D CIM General Purpose Matrix Computation Using 3D Integrated SRAM eDRAM Hybrid Compute In Memory on Memory Architecture](https://arxiv.org/abs/2604.13969)
*Subhradip Chakraborty,Ankur Singh,Akhilesh R. Jaiswal*

Main category: cs.AR

TL;DR: 提出一种3D集成的SRAM-eDRAM混合存内计算架构，支持通用矩阵运算（转置、逐元素加减乘等），突破传统存内计算仅支持点积运算的限制。


<details>
  <summary>Details</summary>
Motivation: 传统存内计算架构主要局限于点积运算，难以高效支持通用矩阵运算（如转置、逐元素加减乘等），限制了存内计算在更广泛AI加速和通用高性能计算中的应用。

Method: 采用3D集成的SRAM-eDRAM混合存内计算架构，基于转置架构设计，支持存内算术运算，采用外围感知设计，在GlobalFoundries 22nm FDSOI工艺上实现4位精度计算。

Result: 该架构能够在内存交叉开关中直接执行通用矩阵运算，在延迟、能效和计算密度之间取得平衡，同时保持与传统存内计算点积架构的兼容性。

Conclusion: 这种内存对内存的存内计算框架将存内计算从点积运算扩展到通用矩阵处理，为AI加速和通用高性能计算的更广泛应用铺平了道路。

Abstract: With the rapid growth of deep neural networks (DNNs), compute-in-memory (CIM) has emerged as a promising energy-efficient paradigm for accelerating multiply-and-accumulate (MAC) operations. Yet, current CIM architectures are largely limited to dot-product computations and struggle to efficiently support general-purpose matrix operations, such as transpose, element-wise addition, and multiplication. This work presents a 3D-integrated, memory-on-memory SRAM-eDRAM hybrid CIM architecture, implemented in GlobalFoundries 22~nm FDSOI technology, capable of performing general matrix operations directly within the memory crossbar with 4-bit precision. By leveraging a specialized transpose-based architecture, in-memory arithmetic operations, peripheral-aware design, and 3D SRAM--eDRAM integration, the proposed architecture balances latency, energy efficiency, and compute density for general purpose matrix operations while remaining compatible with the conventional CIM dot product architectures. Overall, this memory-on-memory CIM framework generalizes CIM beyond dot products, enabling versatile matrix processing and paving the way for broader applications in AI acceleration and general-purpose high performance computing.

### [5] [An ASIC Emulated Oscillator Ising/Potts Machine Solving Combinatorial Optimization Problems](https://arxiv.org/abs/2604.14027)
*Yilmaz Ege Gonul,Baris Taskin*

Main category: cs.AR

TL;DR: 提出一种基于ASIC的数字振荡器伊辛/波茨机架构，通过固定点Kuramoto模型模拟耦合振荡器同步动力学，解决组合优化问题，相比模拟实现具有更高精度和可编程性，相比GPU/CPU模拟具有更高能效和速度。


<details>
  <summary>Details</summary>
Motivation: 模拟振荡器伊辛/波茨机（OIMs/OPMs）虽然速度快但存在耦合分辨率有限、工艺变化敏感和可扩展性问题；数字GPU/CPU模拟虽然灵活但存在不规则内存访问模式和能效低下问题。需要一种结合两者优点的解决方案。

Method: 设计定制ASIC架构，使用简化的固定点Kuramoto模型方程数字模拟OIM/OPM动力学。采用可扩展设计，处理单元之间直接互连，消除共享内存瓶颈，同时保持数字可编程性和精度。原型实现20x20处理单元阵列，采用王图连接拓扑。

Result: 通过后布局仿真评估非加权/加权最大割和图着色问题，达到97-100%的最大准确率，相比通用计算平台在速度和能效方面有显著提升。

Conclusion: 该工作展示了算法协同设计的ASIC在解决NP难组合优化问题方面的可行性，为高性能、高能效的专用硬件加速器提供了有效方案。

Abstract: Oscillator-based Ising/Potts machines (OIMs/OPMs) are promising hardware accelerators for NP-hard combinatorial optimization problems using coupled oscillator synchronization dynamics. Analog OIMs/OPMs offer speed advantages but have limited coupling resolution, process variation susceptibility, and scalability issues, while digital GPU/CPU emulations provide flexibility but suffer from irregular memory access patterns and energy inefficiency. This work presents a custom ASIC architecture that digitally emulates OIM/OPM dynamics using simplified fixedpoint Kuramoto model equations. The scalable design features processing elements with direct interconnections, eliminating shared memory bottleneck while maintaining digital programmability and precision. A 20x20 processing element array with king's graph connectivity is prototyped and evaluated via post-layout simulations on unweighted/weighted max-cut and graph coloring problems, achieving 97-100% maximum accuracy with significant speed and energy improvements over general-purpose platforms, demonstrating the viability of algorithmically codesigned ASICs.

</details>
