### [1] [DART: Input-Difficulty-AwaRe Adaptive Threshold for Early-Exit DNNs](https://arxiv.org/abs/2603.12269)
*Parth Patne,Mahdi Taheri,Christian Herglotz,Maksim Jenihhin,Milos Krstic,Michael Hübner*

Main category: cs.AR

TL;DR: DART框架通过轻量级难度估计、联合退出策略优化和自适应系数管理，实现输入难度感知的自适应推理，在多种DNN上获得显著加速和能效提升。


<details>
  <summary>Details</summary>
Motivation: 现有早期退出方法存在退出策略次优、忽略输入难度、阈值独立优化等问题，需要更智能的自适应推理框架来提升边缘AI加速器的资源效率。

Method: 提出DART框架：1) 轻量级难度估计模块量化输入复杂度；2) 基于动态规划的联合退出策略优化算法；3) 自适应系数管理系统；并引入DAES多目标评估指标。

Result: 在AlexNet、ResNet-18、VGG-16上实现3.3倍加速、5.1倍能耗降低、42%平均功耗降低；扩展到ViT(LeViT)获得5.0倍功耗降低和3.6倍执行时间提升；DAES指标上比基线提升14.8倍。

Conclusion: DART框架显著提升早期退出网络的效率-准确性权衡，但ViT上的精度损失表明需要专门的transformer早期退出机制，DAES为多目标评估提供了新标准。

Abstract: Early-exit deep neural networks enable adaptive inference by terminating computation when sufficient confidence is achieved, reducing cost for edge AI accelerators in resource-constrained settings. Existing methods, however, rely on suboptimal exit policies, ignore input difficulty, and optimize thresholds independently. This paper introduces DART (Input-Difficulty-Aware Adaptive Threshold), a framework that overcomes these limitations. DART introduces three key innovations: (1) a lightweight difficulty estimation module that quantifies input complexity with minimal computational overhead, (2) a joint exit policy optimization algorithm based on dynamic programming, and (3) an adaptive coefficient management system. Experiments on diverse DNN benchmarks (AlexNet, ResNet-18, VGG-16) demonstrate that DART achieves up to \textbf{3.3$\times$} speedup, \textbf{5.1$\times$} lower energy, and up to \textbf{42\%} lower average power compared to static networks, while preserving competitive accuracy. Extending DART to Vision Transformers (LeViT) yields power (5.0$\times$) and execution-time (3.6$\times$) gains but also accuracy loss (up to 17 percent), underscoring the need for transformer-specific early-exit mechanisms. We further introduce the Difficulty-Aware Efficiency Score (DAES), a novel multi-objective metric, under which DART achieves up to a 14.8 improvement over baselines, highlighting superior accuracy, efficiency, and robustness trade-offs.

### [2] [HyperCroc: End-to-End Open-Source RISC-V MCU with a Plug-In Interface for Domain-Specific Accelerators](https://arxiv.org/abs/2603.12308)
*Philippe Sauter,Thomas Benz,Paul Scheffler,Luca Benini*

Main category: cs.AR

TL;DR: HyperCroc扩展了开源RISC-V Croc SoC，集成HyperBus控制器和DMA引擎，为MCU级平台提供高效外部存储访问和加速器支持。


<details>
  <summary>Details</summary>
Motivation: 当前开源微控制器缺乏高效的大规模数据移动和高带宽访问能力，而机器学习和信号处理等应用需要这些能力来支持领域特定加速器。

Method: 扩展Croc SoC，集成硅验证的HyperBus控制器（支持外部DRAM和Flash）和DMA引擎，保持开源综合和物理实现流程，使用IHP 130nm PDK。

Result: HyperBus提供高达400MB/s的低引脚数PSDRAM接口，DMA支持无CPU干预的高吞吐传输。芯片可在消费级工作站1小时内完成实现，首次流片验证72MHz@1.2V下完全功能。

Conclusion: HyperCroc为开源MCU平台提供了实用的领域特定加速支持，通过硅验证的HyperBus和DMA实现了高效数据访问，验证了端到端开源设计流程。

Abstract: Domain-Specific architectures with accelerators for machine learning and signal processing require efficient bulk data movement and high-bandwidth access to large datasets. Such capabilities are often absent from minimal open-source microcontrollers (MCUs). We present HyperCroc, an extension to the end-to-end open-source RISC-V Croc system-on-chip (SoC) integrating a silicon-proven HyperBus controller for off-chip DRAM and Flash memory access and a DMA engine, providing a practical MCU-class platform with streamlined plug-in support for domain-specific acceleration. HyperBus offers a low-pin-count PSDRAM interface at up to 400 MB/s, enabling bandwidth-scaled dataset access, while the DMA engine enables autonomous, high-throughput transfers without CPU intervention. HyperCroc preserves Croc's open-source synthesis and physical implementation flow targeting IHP's open 130 nm process design kit (PDK); the full chip can be implemented in under one hour on a consumer-grade workstation. We further report first silicon measurements from MLEM, the first Croc tapeout, confirming that the silicon is fully functional at 72 MHz @ 1.2 V and validating the end-to-end flow.

### [3] [DiscoRD: An Experimental Methodology for Quickly Discovering the Reliable Read Disturbance Threshold of Real DRAM Chips](https://arxiv.org/abs/2603.12435)
*Ataberk Olgun,F. Nisa Bostanci,Ismail Emir Yuksel,Haocong Luo,Minesh Patel,A. Giray Yaglikci,Onur Mutlu*

Main category: cs.AR

TL;DR: DiscoRD：一种快速可靠的DRAM读干扰测试方法，通过实验建模和配置机制降低不可纠正错误概率


<details>
  <summary>Details</summary>
Motivation: 现有DRAM读干扰缓解技术依赖读干扰阈值(RDT)，但全面准确测量每个DRAM行的RDT耗时严重。需要开发快速可靠的测试方法来支持安全、高性能和能效的系统设计。

Method: 基于212个真实DDR4芯片的广泛实验表征研究，测量数十万DRAM行数百万次RDT。开发读干扰比特翻转的经验模型，评估使用单一RDT_min配置时的不可纠正错误概率。结合ECC、内存清理和可配置读干扰缓解机制。

Result: 1) 仅依赖轻量级ECC会产生相对较高的不可纠正错误概率；2) 结合ECC、不定期内存清理和可配置读干扰缓解机制可大幅降低错误概率；3) 可更精确识别每个独立行的RDT。

Conclusion: 容错、内存清理、在线分析和运行时可配置的读干扰缓解技术对于实现安全、能效且考虑空间变化的读干扰缓解至关重要。DiscoRD有助于在性能/成本与可靠性之间进行量化权衡。

Abstract: State-of-the-art DRAM read disturbance mitigations rely on the read disturbance threshold (RDT) (e.g., the number of aggressor row activations needed to induce the first read disturbance bitflip) to securely and performance- and energy-efficiently prevent read disturbance bitflips. However, accurately and exhaustively characterizing the RDT of every DRAM row in a chip is time intensive. Rapidly determining RDT is important for enabling secure, performance- and energy-efficient systems. Our goal is to develop and evaluate a reliable and rapid read disturbance testing methodology. To that end, we develop DiscoRD building on the key results of an extensive experimental characterization study using 212 real DDR4 chips whereby we measure the RDT of hundreds of thousands of DRAM rows millions of times.
  We develop an empirical model for read disturbance bitflips and evaluate the probability of read-disturbance-induced uncorrectable errors when a read disturbance mechanism is configured using a single $RDT_{min}$ measurement. Using this model we demonstrate that 1) relying on a lightweight error-correcting code (ECC) alone yields relatively high uncorrectable error probability and 2) combining ECC, infrequent memory scrubbing, and configurable read disturbance mitigation mechanisms can greatly reduce the error probability. Building on our observations and analyses, we discuss the RDT of each individual row can be identified more precisely. Our results show that error tolerance, memory scrubbing, online profiling, and run-time configurable read disturbance mitigation techniques are important to enable secure and energy-efficient spatial-variation aware read disturbance mitigations. We hope that DiscoRD drives research that enables us to quantitatively navigate the performance/cost - reliability tradeoff space for read disturbance mitigation techniques.

### [4] [System-Technology Co-Optimization of Bitline Routing and Bonding Pathways in Monolithic 3D DRAM Architectures](https://arxiv.org/abs/2603.12461)
*Kiseok Lee,Sungwon Cho,Seongkwang Lim,Suman Datta,Shimeng Yu*

Main category: cs.AR

TL;DR: 3D DRAM面临布线限制，采用AOS选择器和位线带架构优化设计，实现2.6 Gb/mm²密度、10.5 ns延迟和60%能耗降低


<details>
  <summary>Details</summary>
Motivation: 3D DRAM虽然能提高密度，但受限于外围电路布线和混合键合约束，可能导致传感裕度、延迟和阵列效率下降，需要优化设计解决这些问题

Method: 使用TCAD提取器件特性和阵列寄生参数，在CMOS键合阵列架构中进行SPICE仿真，采用非晶氧化物半导体选择器和位线带架构管理布线拥塞和寄生效应

Result: 优化设计实现2.6 Gb/mm²比特密度（硅存取晶体管137层或AOS 87层），比D1b 2D DRAM密度提高约6倍；行周期时间10.5 ns（D1b为21.3 ns）；读写能耗降低60%

Conclusion: 位线带架构结合AOS选择器是管理3D DRAM布线拥塞和寄生效应的关键，能显著提高密度、降低延迟和能耗，为3D DRAM发展提供可行方案

Abstract: 3D DRAM has emerged as a promising approach for continued density scaling, but its viability is limited by routing and hybrid bonding constraints to periphery, which may degrade sensing margin, latency, and array efficiency. With device characteristics and array parasitics extracted from TCAD, SPICE simulations are performed with peri logic in a CMOS-Bonded-Array (CBA). The analysis shows that the bitline strap architecture with amorphous oxide semiconductor (AOS) selectors is essential to manage routing congestion and parasitics. The optimized design achieves a bit density of 2.6 Gb/mm^2 (137 layers with Si access transistors or 87 layers with AOS), representing ~6x density scaling over D1b 2D DRAM. The design further demonstrates a nominal row cycle time (tRC) of 10.5 ns, compared to 21.3 ns in D1b, and a 60% reduction in read/write energy.

### [5] [CellE: Automated Standard Cell Library Extension via Equality Saturation](https://arxiv.org/abs/2603.12797)
*Yi Ren,Yukun Wang,Xiang Meng,Guoyao Cheng,Baokang Peng,Lining Zhang,Yibo Lin,Runsheng Wang,Guangyu Sun*

Main category: cs.AR

TL;DR: CellE是一个利用形式化方法实现功能等价子电路穷举发现的自动化标准单元库扩展框架，通过等式饱和和模式挖掘实现面积优化，实验显示平均面积减少15.41%，延迟减少8.00%。


<details>
  <summary>Details</summary>
Motivation: 自动化标准单元库扩展对于最大化现代VLSI设计的QoR至关重要。现有方法在发现功能等价子电路方面存在局限性，需要更系统的方法来穷举发现等价实现以优化面积和性能。

Method: CellE框架利用形式化方法，对后映射网表应用等式饱和生成e-graph，聚类所有功能等价实现。通过这种规范表示，采用高效的模式挖掘算法选择面积最优的标准单元。

Result: 实验结果显示平均面积减少15.41%（相比先前工作最高达23.64%）。在商业流程中的表征显示平均延迟减少8.00%，证实了CellE在QoR优化方面的优越能力。

Conclusion: CellE通过形式化方法和等式饱和实现了功能等价子电路的穷举发现，显著改善了标准单元库扩展的QoR优化能力，为VLSI设计提供了有效的面积和延迟优化方案。

Abstract: Automated standard cell library extension is crucial for maximizing Quality of Results (QoR) in modern VLSI design. We introduce CellE, a novel framework that leverages formal methods to achieve exhaustive discovery of functionally equivalent subcircuits. CellE applies equality saturation to the post-mapping netlist, generating an e-graph to cluster all functionally equivalent implementations. This canonical representation enables an efficient pattern mining algorithm to select the most area-optimal standard cells. Experimental results show a 15.41% average area reduction (up to 23.64% over prior work). Furthermore, characterization in a commercial flow demonstrates an 8.00% average delay reduction, confirming CellE's superior QoR optimization capabilities.

</details>
