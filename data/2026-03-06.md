### [1] [HDLFORGE: A Two-Stage Multi-Agent Framework for Efficient Verilog Code Generation with Adaptive Model Escalation](https://arxiv.org/abs/2603.04646)
*Armin Abdollahi,Saeid Shokoufa,Negin Ashrafi,Mehdi Kamal,Massoud Pedram*

Main category: cs.AR

TL;DR: HDLFORGE是一个两阶段多智能体框架，用于自动化Verilog生成，通过智能升级策略在生成速度和准确性之间取得优化平衡。


<details>
  <summary>Details</summary>
Motivation: 现有的Verilog生成系统在生成速度和准确性之间存在权衡问题，要么使用小型模型速度快但准确性低，要么使用大型模型准确但速度慢且成本高。

Method: 采用两阶段多智能体框架：第一阶段使用中型LLM的紧凑编码器，第二阶段仅在需要时升级到超大型LLM的更强编码器。关键创新包括基于反例引导的形式化智能体，将有界模型检查跟踪转换为可重用微测试，以及可移植的升级控制器。

Result: 在VerilogEval Human、VerilogEval V2和RTLLM基准测试中，HDLFORGE-Qwen分别达到91.2%、91.8%和97.2%的Pass@1/Pass@5准确率，中位延迟降低约50%，显著优于其他中型模型。

Conclusion: HDLFORGE通过智能的两阶段升级策略，在保持高准确性的同时显著降低延迟，为硬件描述语言生成提供了高效的自动化解决方案。

Abstract: We present HDLFORGE, a two-stage multi-agent framework for automated Verilog generation that optimizes the trade-off between generation speed and accuracy. The system uses a compact coder with a medium-sized LLM by default (Stage A) and escalates to a stronger coder with an ultra-large LLM (Stage B) only when needed, guided by a calibrated score from inexpensive diagnostics including compilation, lint, and smoke tests. A key innovation is a counterexample-guided formal agent that converts bounded-model-checking traces into reusable micro-tests, significantly reducing bug detection time and repair iterations. The portable escalation controller can wrap existing Verilog LLM pipelines without modifying their internals. Evaluated on VerilogEval Human, VerilogEval V2, and RTLLM benchmarks, HDLFORGE demonstrates improved accuracy-latency trade-offs compared to single-stage systems through comprehensive analysis of wall-clock time distributions, escalation thresholds, and agent ablations. On VerilogEval Human and VerilogEval V2, HDLFORGE-Qwen achieves 91.2% and 91.8% Pass@1 with roughly 50% lower median latency, dramatically improving accuracy over other medium-sized models, and 97.2% Pass@5 on RTLLM.

### [2] [Hardware-Software Co-design for 3D-DRAM-based LLM Serving Accelerator](https://arxiv.org/abs/2603.04797)
*Cong Li,Yihan Yin,Chenhao Xue,Zhao Wang,Fujun Bai,Yixin Guo,Xiping Jiang,Qiang Wu,Yuan Xie,Guangyu Sun*

Main category: cs.AR

TL;DR: Helios是一个基于混合键合技术的LLM服务加速器，通过优化KV缓存管理和分布式注意力执行，显著提升动态LLM服务的性能和能效。


<details>
  <summary>Details</summary>
Motivation: 现有近内存处理(NMP)设计采用粗粒度的KV缓存管理和不灵活的注意力执行流程，无法高效处理高度动态的LLM服务工作负载，限制了LLM服务的加速能力。

Method: 设计了Helios混合键合LLM服务加速器，包括：1）PE内部执行流程和PE间通信原语，支持分布式分块注意力执行；2）空间感知KV缓存分配机制，平衡注意力工作负载分布并最小化PE间数据传输开销。

Result: 与现有GPU/NMP设计相比，Helios实现了3.25倍（几何平均）加速和3.36倍（几何平均）能效提升，同时将P50/P99 token间时间延迟降低高达72%/76%。

Conclusion: Helios通过解决KV缓存管理与分布式内存抽象之间的根本差距，为动态LLM服务提供了高效的加速解决方案。

Abstract: Large language models (LLMs) have been widely deployed for online generative services, where numerous LLM instances jointly handle workloads with fluctuating request arrival rates and variable request lengths. To efficiently execute coexisting compute-intensive and memory-intensive operators, near-memory processing (NMP) based computing paradigm has been extensively proposed. However, existing NMP designs adopt coarse-grained KV cache management and inflexible attention execution flow. Such limitations hinder these proposals from efficiently handling \textit{highly dynamic} LLM serving workloads, limiting their ability to accelerate LLM serving.
  To tackle these problems, we propose Helios, a Hybrid-bonding-based \uline{L}LM \uline{S}erving accelerator. Helios aims to bridge the fundamental gap between the dynamic nature of KV cache management in LLM serving and the distributed, non-uniform memory abstraction among NMP processing engines (PEs). To this end, we design both the intra-PE execution flow and the inter-PE communication primitives for distributed tiled attention execution. We further propose \textit{spatially-aware} KV cache allocation mechanism to balance the attention workload distribution while minimizing the inter-PE data transfer overhead. Compared with existing GPU/NMP designs, Helios achieves 3.25 times (geomean) speedup and 3.36 times (geomean) better energy efficiency, along with up to 72%/76% P50/P99 time-between-tokens degradation.

### [3] [VMXDOTP: A RISC-V Vector ISA Extension for Efficient Microscaling (MX) Format Acceleration](https://arxiv.org/abs/2603.04979)
*Max Wipfli,Gamze İslamoğlu,Navaneeth Kunhi Purayil,Angelo Garofalo,Luca Benini*

Main category: cs.AR

TL;DR: 提出VMXDOTP指令集扩展，用于高效执行微缩放格式的点积运算，解决MX格式与向量执行不匹配的问题，在12nm工艺下实现高能效的MX计算加速。


<details>
  <summary>Details</summary>
Motivation: 现代transformer模型需要灵活加速器，微缩放格式虽能减少数据量但语义与向量执行不匹配，导致计算资源利用率低和性能下降。

Method: 提出VMXDOTP RISC-V向量指令集扩展，支持MXFP8和MXFP4输入、FP32/BF16累加、软件定义块大小，在共享L1内存向量处理单元集群中实现。

Result: 在12nm FinFET工艺中实现，达到125 MXFP8-GFLOPS和250 MXFP4-GFLOPS，能效843/1632 GFLOPS/W，面积开销仅7.2%，相比软件模拟获得7.0倍加速和4.9倍能效提升。

Conclusion: VMXDOTP有效解决了微缩放格式与向量执行的匹配问题，支持可变块大小，在面积效率和能效方面优于现有MX引擎，为现代transformer模型提供了高效的硬件加速方案。

Abstract: Compared to the first generation of deep neural networks, dominated by regular, compute-intensive kernels such as matrix multiplications (MatMuls) and convolutions, modern decoder-based transformers interleave attention, normalization, and data-dependent control flow. This demands flexible accelerators, a requirement met by scalable, highly energy-efficient shared-L1-memory vector processing element (VPE) clusters. Meanwhile, the ever-growing size and bandwidth needs of state-of-the-art models make reduced-precision formats increasingly attractive. Microscaling (MX) data formats, based on block floating-point (BFP) representations, have emerged as a promising solution to reduce data volumes while preserving accuracy. However, MX semantics are poorly aligned with vector execution: block scaling and multi-step mixed-precision operations break the regularity of vector pipelines, leading to underutilized compute resources and performance degradation. To address these challenges, we propose VMXDOTP, a RISC-V Vector (RVV) 1.0 instruction set architecture (ISA) extension for efficient MX dot product execution, supporting MXFP8 and MXFP4 inputs, FP32 and BF16 accumulation, and software-defined block sizes. A VMXDOTP-enhanced VPE cluster achieves up to 97 % utilization on MX-MatMul. Implemented in 12 nm FinFET, it achieves up to 125 MXFP8-GFLOPS and 250 MXFP4-GFLOPS, with 843/1632 MXFP8/MXFP4-GFLOPS/W at 1 GHz, 0.8 V, and only 7.2 % area overhead. Our design yields up to 7.0x speedup and 4.9x energy efficiency with respect to software-emulated MXFP8-MatMul. Compared with prior MX engines, VMXDOTP supports variable block sizes, is up to 1.4x more area-efficient, and delivers up to 2.1x higher energy efficiency.

### [4] [Network Design for Wafer-Scale Systems with Wafer-on-Wafer Hybrid Bonding](https://arxiv.org/abs/2603.05266)
*Patrick Iff,Tommaso Bonato,Maciej Besta,Luca Benini,Torsten Hoefler*

Main category: cs.AR

TL;DR: 研究晶圆级集成中晶片布局对网络拓扑和通信性能的影响，提出了四种布局方案，显著提升吞吐量、降低延迟和能耗。


<details>
  <summary>Details</summary>
Motivation: 随着Transformer大语言模型的发展，数据移动成为主要瓶颈，特别是芯片边界通信带宽急剧下降。晶圆级集成通过晶圆间混合键合提供超高带宽，但晶片物理布局会影响网络拓扑和通信性能。

Method: 从2D网状基线开始，提出了四种晶片布局方案：对齐布局、交错布局、旋转布局和轮廓布局，分析它们对网络拓扑和通信性能的影响。

Result: 相比基线，提出的四种布局方案能提升吞吐量最高达250%，降低延迟最高达36%，减少每字节传输能耗最高达38%。

Conclusion: 晶片物理布局对晶圆级集成的网络拓扑和通信性能有显著影响，优化布局设计可以大幅提升系统性能，缓解大语言模型的数据移动瓶颈。

Abstract: Transformer-based large language models are increasingly constrained by data movement as communication bandwidth drops sharply beyond the chip boundary. Wafer-scale integration using wafer-on-wafer hybrid bonding alleviates this limitation by providing ultra-high bandwidth between reticles on bonded wafers. In this paper, we investigate how the physical placement of reticles on wafers influences the achievable network topology and the resulting communication performance. Starting from a 2D mesh-like baseline, we propose four reticle placements (Aligned, Interleaved, Rotated, and Contoured) that improve throughput by up to 250%, reduce latency by up to 36%, and decrease energy per transmitted byte by up to 38%.

### [5] [NL2GDS: LLM-aided interface for Open Source Chip Design](https://arxiv.org/abs/2603.05489)
*Max Eland,Jeyan Thiyagalingam,Dinesh Pamunuwa,Roshan Weerasekera*

Main category: cs.AR

TL;DR: NL2GDS是一个利用大语言模型将自然语言硬件描述转换为可综合RTL和完整GDSII版图的新框架，通过OpenLane ASIC流程实现，在基准测试中实现了显著的性能提升。


<details>
  <summary>Details</summary>
Motivation: 硬件设计复杂性日益增加，高层次规范与RTL实现之间的差距阻碍了快速原型设计和系统设计，需要更高效的设计方法。

Method: 采用模块化流水线：捕获非正式设计意图，使用多个LLM引擎生成HDL并进行验证，通过OpenLane ASIC流程编排自动化综合和版图生成。

Result: 在ISCAS'85和ISCAS'89基准设计上评估，相比基线设计实现了最高36%的面积减少、35%的延迟减少和70%的功耗节省。

Conclusion: NL2GDS框架展示了利用LLM实现从自然语言到完整ASIC设计的潜力，有望民主化ASIC设计并加速硬件创新。

Abstract: The growing complexity of hardware design and the widening gap between high-level specifications and register-transfer level (RTL) implementation hinder rapid prototyping and system design. We introduce NL2GDS (Natural Language to Layout), a novel framework that leverages large language models (LLMs) to translate natural language hardware descriptions into synthesizable RTL and complete GDSII layouts via the open-source OpenLane ASIC flow. NL2GDS employs a modular pipeline that captures informal design intent, generates HDL using multiple LLM engines and verifies them, and orchestrates automated synthesis and layout. Evaluations on ISCAS'85 and ISCAS'89 benchmark designs demonstrate up to 36% area reduction, 35% delay reduction, and 70% power savings compared to baseline designs, highlighting its potential to democratize ASIC design and accelerate hardware innovation.

</details>
