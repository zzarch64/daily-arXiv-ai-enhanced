### [1] [ChatSVA: Bridging SVA Generation for Hardware Verification via Task-Specific LLMs](https://arxiv.org/abs/2604.02811)
*Lik Tung Fu,Jie Zhou,Shaokai Ren,Mengli Zhang,Jia Xiong,Hugo Jiang,Nan Guan,Xi Wang,Jun Yang*

Main category: cs.AR

TL;DR: ChatSVA是一个基于多智能体框架的端到端SystemVerilog断言生成系统，通过AgentBridge平台生成高纯度数据集解决领域数据稀缺问题，在24个RTL设计上实现了98.66%语法正确率和96.12%功能正确率。


<details>
  <summary>Details</summary>
Motivation: 功能验证消耗超过50%的IC开发生命周期，手动编写SystemVerilog断言(SVAs)既费时又容易出错。虽然大型语言模型(LLMs)有潜力，但直接部署面临功能准确性低和领域特定数据严重稀缺的挑战。

Method: 引入ChatSVA，一个基于多智能体框架的端到端SVA生成系统。核心是AgentBridge平台，通过系统化生成高纯度数据集来克服少样本场景中的数据稀缺问题。

Result: 在24个RTL设计上评估，ChatSVA实现了98.66%的语法通过率和96.12%的功能通过率，每个设计生成139.5个SVAs，功能覆盖率达到82.50%。相比之前的最先进技术，功能正确性提高了33.3个百分点，功能覆盖率提升了超过11倍。

Conclusion: ChatSVA不仅在自动化SVA生成方面设定了新的最先进水平，还为解决少样本、领域特定场景中的长链推理问题建立了稳健的框架。已公开发布在线服务。

Abstract: Functional verification consumes over 50% of the IC development lifecycle, where SystemVerilog Assertions (SVAs) are indispensable for formal property verification and enhanced simulation-based debugging. However, manual SVA authoring is labor-intensive and error-prone. While Large Language Models (LLMs) show promise, their direct deployment is hindered by low functional accuracy and a severe scarcity of domain-specific data. To address these challenges, we introduce ChatSVA, an end-to-end SVA generation system built upon a multi-agent framework. At its core, the AgentBridge platform enables this multi-agent approach by systematically generating high-purity datasets, overcoming the data scarcity inherent to few-shot scenarios. Evaluated on 24 RTL designs, ChatSVA achieves 98.66% syntax and 96.12% functional pass rates, generating 139.5 SVAs per design with 82.50% function coverage. This represents a 33.3 percentage point improvement in functional correctness and an over 11x enhancement in function coverage compared to the previous state-of-the-art (SOTA). ChatSVA not only sets a new SOTA in automated SVA generation but also establishes a robust framework for solving long-chain reasoning problems in few-shot, domain-specific scenarios. An online service has been publicly released at https://www.nctieda.com/CHATDV.html.

### [2] [EEspice: A Modular Circuit Simulation Platform with Parallel Device Model Evaluation via Graph Coloring](https://arxiv.org/abs/2604.03079)
*Xuanhao Bao,Danial Chitnis*

Main category: cs.AR

TL;DR: EEspice是一个开源电路仿真框架，通过解耦器件模型评估为可独立替换的内核，采用并行stamping策略解决多核竞争更新系统矩阵的瓶颈问题，在64核工作站上实现最高45倍加速。


<details>
  <summary>Details</summary>
Motivation: 现代模拟/混合信号设计越来越依赖优化循环流程，如基于AI和LLM的尺寸优化代理需要反复调用高效准确的SPICE仿真器。然而，非线性模型（特别是BSIM模型）的计算成本仍然是主要瓶颈。标准并行化方法中，晶体管等器件可以轻松分配到不同处理器，但后续的stamping阶段（各器件贡献添加到共享系统矩阵）常成为瓶颈，因为多核同时更新相同矩阵元素会导致竞争，系统被迫串行处理任务。

Method: EEspice采用模块化架构，将器件模型评估解耦为可独立替换的内核，实现并行stamping策略。具体方法是将MOSFET实例划分为独立的颜色组，这些组可以并行处理，避免了多核竞争更新同一矩阵元素的问题。

Result: 在64核工作站上，当冲突较低时，该方法相比单线程性能实现了最高45倍的加速。分析还探讨了性能如何依赖于电路拓扑结构。

Conclusion: EEspice通过创新的并行stamping策略有效解决了电路仿真中的多核竞争瓶颈，显著提升了仿真速度，为现代IC设计优化流程提供了更高效的仿真基础。

Abstract: As modern analogue/mixed-signal design increasingly relies on optimization-in-the-loop flows, such as AI and LLM-based sizing agents that repeatedly invoke SPICE-efficient, accurate high-performance simulators have become an indispensable foundation for modern integrated circuit (IC) design. However, the computational cost of evaluating nonlinear models, particularly for BSIM models, remains a significant bottleneck. In standard parallelization approaches, devices such as transistors are easily distributed across processors. The subsequent stamping phase, where each device's contributions are added to the shared system matrix, often creates a bottleneck. Because multiple processor cores compete to update the same matrix elements simultaneously, the system is forced to process tasks one at a time to avoid errors. This paper introduces EEspice, an open-source circuit simulation framework whose modular architecture decouples device model evaluation into independently replaceable kernels, enabling a parallel stamping strategy that overcomes this bottleneck. It partitions MOSFET instances into independent color groups, which can be processed in parallel. Our results show that on a 64-core workstation, the proposed approach achieves up to 45x speedup over single-thread performance when conflicts are low. Our analysis also explores how performance depends on circuit topology.

Main category: cs.AR

TL;DR: InCoder-32B-Thinking：通过ECoT合成框架和工业代码世界模型生成推理轨迹，在工业软件任务中实现最先进的开源性能

Motivation: 工业软件开发（芯片设计、GPU优化、嵌入式系统）缺乏专家推理轨迹，无法展示工程师如何考虑硬件约束和时序语义

Method: 1. 使用ECoT框架通过多轮对话和环境错误反馈合成推理链，显式建模错误修正过程；2. 训练工业代码世界模型学习领域特定执行轨迹，理解代码如何影响硬件行为的因果动态；3. 所有合成推理轨迹通过领域工具链验证，创建匹配工业任务自然推理深度分布的训练数据

Abstract: Industrial software development across chip design, GPU optimization, and embedded systems lacks expert reasoning traces showing how engineers reason about hardware constraints and timing semantics. In this work, we propose InCoder-32B-Thinking, trained on the data from the Error-driven Chain-of-Thought (ECoT) synthesis framework with an industrial code world model (ICWM) to generate reasoning traces. Specifically, ECoT generates reasoning chains by synthesizing the thinking content from multi-turn dialogue with environmental error feedback, explicitly modeling the error-correction process. ICWM is trained on domain-specific execution traces from Verilog simulation, GPU profiling, etc., learns the causal dynamics of how code affects hardware behavior, and enables self-verification by predicting execution outcomes before actual compilation. All synthesized reasoning traces are validated through domain toolchains, creating training data matching the natural reasoning depth distribution of industrial tasks.
