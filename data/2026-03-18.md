### [1] [DRCY: Agentic Hardware Design Reviews](https://arxiv.org/abs/2603.15672)
*Kyle Dumont,Nicholas Herbert,Hayder Tirmazi,Shrikanth Upadhayaya*

Main category: cs.AR

TL;DR: DRCY是一个基于多智能体LLM的生产级系统，用于自动化硬件原理图连接审查，通过自主获取元件数据手册、进行引脚级分析，并在设计评审中发布内联评论。


<details>
  <summary>Details</summary>
Motivation: 硬件设计错误在制造后发现需要昂贵的物理重制，可能导致产品延迟数月。现有EDA工具只能检查结构连接规则，无法验证连接在语义上是否符合元件数据手册规范。

Method: 采用五智能体流水线架构，包括自主数据手册检索系统（带自我评估）和多轮共识机制，用于提高安全关键分析的可靠性。系统在AllSpice Hub平台上作为CI/CD操作部署，在提交设计评审时触发。

Result: DRCY已在AllSpice Hub生产环境中部署，被主要硬件公司定期使用，应用范围涵盖多智能体车辆设计到太空探索等多个用例。

Conclusion: DRCY是首个生产就绪的多智能体LLM系统，能够自动化第一轮原理图连接审查，填补了现有EDA工具在语义验证方面的空白。

Abstract: Hardware design errors discovered after fabrication require costly physical respins that can delay products by months. Existing electronic design automation (EDA) tools enforce structural connectivity rules. However, they cannot verify that connections are \emph{semantically} correct with respect to component datasheets. For example, that a symbol's pinout matches the manufacturer's specification, or that a voltage regulator's feedback resistors produce the intended output. We present DRCY, the first production-ready multi-agent LLM system that automates first-pass schematic connection review by autonomously fetching component datasheets, performing pin-by-pin analysis against extracted specifications, and posting findings as inline comments on design reviews. DRCY is deployed in production on AllSpice Hub, a collaborative hardware design platform, where it runs as a CI/CD action triggered on design review submissions. DRCY is used regularly by major hardware companies for use-cases ranging from multi-agent vehicle design to space exploration. We describe DRCY's five-agent pipeline architecture, its agentic datasheet retrieval system with self-evaluation, and its multi-run consensus mechanism for improving reliability on safety-critical analyses

### [2] [GLANCE: Gaze-Led Attention Network for Compressed Edge-inference](https://arxiv.org/abs/2603.15717)
*Neeraj Solanki,Hong Ding,Sepehr Tabrizchi,Ali Shafiee Sarvestani,Shaahin Angizi,David Z. Pan,Arman Roohi*

Main category: cs.AR

TL;DR: 提出基于生物中央凹视觉启发的两阶段AR/VR实时目标检测系统，使用可微分无权重神经网络进行超高效注视估计，结合注意力引导的感兴趣区域检测，在Arduino Nano上实现亚10ms延迟和显著能效提升。


<details>
  <summary>Details</summary>
Motivation: AR/VR系统对实时目标检测有严格的计算约束，需要在有限功耗预算内实现亚10ms延迟。传统方法计算密集，难以满足可穿戴设备的能效要求。

Method: 采用两阶段流水线：1) 使用可微分无权重神经网络进行注视估计，通过内存查找而非乘累加计算实现超高效跟踪；2) 基于注视预测引导选择性目标检测，仅在注意力区域进行处理，减少计算负担。

Result: 注视估计仅需393次MAC操作和2.2KB内存，角度误差8.32°。系统在Arduino Nano 33 BLE上实现48.1% mAP（注意力区域达51.8%），计算负担减少40-50%，能耗降低65%，延迟亚10ms。相比全局YOLOv12n基线，小/中/大物体检测精度分别从39.2%/63.4%/83.1%提升至51.3%/72.1%/88.1%。

Conclusion: 基于内存中心架构和显式注意力建模的方法在资源受限的可穿戴平台上比均匀处理方法具有更好的效率和精度，满足AR/VR系统的严格实时要求。

Abstract: Real-time object detection in AR/VR systems faces critical computational constraints, requiring sub-10\,ms latency within tight power budgets. Inspired by biological foveal vision, we propose a two-stage pipeline that combines differentiable weightless neural networks for ultra-efficient gaze estimation with attention-guided region-of-interest object detection. Our approach eliminates arithmetic-intensive operations by performing gaze tracking through memory lookups rather than multiply-accumulate computations, achieving an angular error of $8.32^{\circ}$ with only 393 MACs and 2.2 KiB of memory per frame. Gaze predictions guide selective object detection on attended regions, reducing computational burden by 40-50\% and energy consumption by 65\%. Deployed on the Arduino Nano 33 BLE, our system achieves 48.1\% mAP on COCO (51.8\% on attended objects) while maintaining sub-10\,ms latency, meeting stringent AR/VR requirements by improving the communication time by $\times 177$. Compared to the global YOLOv12n baseline, which achieves 39.2\%, 63.4\%, and 83.1\% accuracy for small, MEDium, and LARGE objects, respectively, the ROI-based method yields 51.3\%, 72.1\%, and 88.1\% under the same settings. This work shows that memory-centric architectures with explicit attention modeling offer better efficiency and accuracy for resource-constrained wearable platforms than uniform processing.
