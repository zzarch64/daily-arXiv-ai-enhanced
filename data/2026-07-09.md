### [1] [HiFuzz: Hierarchical Reinforcement Learning for Semantic-Aware and Adaptive CPU Fuzzing](https://arxiv.org/abs/2607.06619)
*Ya Wang,Hanwei Fan,Zhenguo Liu,Xiaofeng Zhou,Yangdi Lyu,Jiang Xu,Wei Zhang*

Main category: cs.AR

TL;DR: HiFuzz是一个新型分层强化学习框架，通过双层生成过程（程序代理负责全局布局，基本块代理负责指令填充）替代传统模糊测试中的突变操作，并引入自适应覆盖奖励机制和语义感知编码器，在RISC-V处理器验证中显著提升了覆盖率和漏洞检测能力。


<details>
  <summary>Details</summary>
Motivation: 传统基于突变的模糊测试在达到处理器深层架构状态时效率低下，需要新的方法来解决这一验证瓶颈。

Method: 提出分层强化学习框架HiFuzz，包含两层生成过程：Program Agent负责全局程序布局，Basic Block Agent负责精确指令填充；同时集成自适应覆盖奖励机制和语义感知的基本块编码器以解决奖励稀疏问题。

Result: 在三个真实RISC-V处理器核上的实验表明，HiFuzz在覆盖率和缺陷检测方面显著优于最先进的模糊测试工具。

Conclusion: 分层强化学习+结构化生成能有效替代传统突变方法，为处理器验证提供更高效的方案。

Abstract: Modern processor verification struggles to reach deep architectural states due to the inefficiencies of traditional mutation-based fuzzing. We propose HiFuzz, a novel hierarchical reinforcement learning framework that replaces mutation with a structured, two-layer generation process: a Program Agent for global layout and a Basic Block Agent for precise instruction filling. To overcome reward sparsity, HiFuzz integrates an adaptive coverage reward mechanism and a semantic-aware basic block encoder providing intrinsic feedback. Extensive evaluations on three real-world RISC-V cores demonstrate that HiFuzz significantly outperforms state-of-the-art fuzzers in coverage and bug detection.

### [2] [ThermoDSE: A Thermal-Aware and Comprehensive Design Space Exploration for Chiplet-Based DNN Accelerators](https://arxiv.org/abs/2607.07096)
*Jian Peng,Hanwei Fan,Jingbo Jiang,Lin Jiang,Wei Zhang*

Main category: cs.AR

TL;DR: 本文提出 ThermoDSE，首个联合考虑细粒度任务建模、芯粒粒度、核心粒度和热/面积约束的芯粒DNN加速器设计空间探索框架，相比Simba等基线实现最高3.5倍EDY³提升，并比模拟退火和强化学习方法分别快3.7倍和29.4倍。


<details>
  <summary>Details</summary>
Motivation: 基于芯粒的DNN加速器面临面积和热约束的严峻挑战，现有设计空间优化未同时考虑细粒度任务建模、芯粒粒度、核心粒度及关键物理约束，缺乏统一框架。

Method: 提出ThermoDSE框架，将细粒度建模技术集成到统一仿真优化框架中，联合优化架构设计、任务编排和芯粒间通信，同时满足严格的热和面积约束。

Result: 相比Simba等基线，EDY³（能量×延迟×良率倒数）最高提升3.5倍；相比模拟退火和强化学习方法，收敛到更优设计点，运行速度分别提升3.7倍和29.4倍。

Conclusion: ThermoDSE是首个全面考虑热/面积约束、多粒度建模的芯粒DNN加速器设计空间探索框架，显著优于现有方法。

Abstract: Chiplet-based DNN accelerators provide a scalable path to balance performance and yield for modern AI workloads. However, such systems face critical challenges in area and thermal constraints. Design space optimization should jointly consider fine-grained task modeling, chiplet granularity, core granularity, and critical physical constraints. To the best of our knowledge, this is the first framework that involves all these factors.
  In this work, we propose ThermoDSE, a thermal-aware and comprehensive design space exploration framework for chiplet-based DNN accelerators. ThermoDSE integrates existing fine-grained modeling techniques into a unified simulation and optimization framework that jointly considers architecture design, task orchestration, and inter-chiplet communication under strict thermal and area constraints. Experimental results show that ThermoDSE achieves up to 3.5x improvement in Energy-Delay-Inverse-Yield, defined as E times D times inverse Y, compared with state-of-the-art Simba and other baselines. Furthermore, relative to simulated annealing and reinforcement learning-based methods, ThermoDSE converges to better design points with 3.7x and 29.4x runtime speedups, respectively.

### [3] [Miter-Aware LUT Mapping: Aligning Structure and Solvability for Efficient Logic Equivalence Checking](https://arxiv.org/abs/2607.07164)
*Jiaying Zhu,Zhengyuan Shi,Mengxia Tao,Kezhi Li,Min Li,Qiang Xu*

Main category: cs.AR

TL;DR: 提出了一种面向逻辑等价性检查的miter感知映射框架，通过构建LUT-based miter来替代传统扁平网表，结合等价保持映射、高斯引导XOR建模和求解器导向LUT选择三种技术，显著提升SAT求解效率，最高减少92.1%的求解时间。


<details>
  <summary>Details</summary>
Motivation: 逻辑等价性检查（LEC）中，综合引入的结构扰动和XOR密集区域严重制约了SAT求解器性能，现有方法过度关注求解器本身而忽视了miter建模的重要性。

Method: 提出miter感知映射框架：(1) 等价保持映射，实现两个电路的结构对齐；(2) 高斯引导XOR建模，代数简化密集算术区域；(3) 求解器导向LUT选择，生成适合SAT推理的优化表示。

Result: 在多个数据集上，该方法使最先进的SAT求解器求解时间最高减少92.1%。

Conclusion: 求解器感知的建模范式——将结构映射与SAT推理统一起来——能够从根本上提升LEC效率。

Abstract: Logic Equivalence Checking (LEC), a fundamental hardware verification task, is often bottlenecked by synthesis-induced structural perturbations and XOR-dense regions that degrade SAT solver performance. We contend that the modeling of the miter is as critical as the SAT solver itself. To this end, we introduce a miter-aware mapping framework that strategically formulates the problem before solving. By constructing a LUT-based miter -- instead of a traditional, flat netlist -- our approach preserves critical structural correspondence between the two designs while making high-level logic relations explicit. Our framework uniquely integrates three techniques: equivalence-preserving mapping to structurally align the two circuits, Gaussian-guided XOR modeling to algebraically simplify dense arithmetic, and solver-oriented LUT selection to generate a representation optimized for efficient SAT reasoning. Evaluated on comprehensive datasets, our method achieves up to a \textbf{92.1\%} reduction across state-of-the-art SAT solvers. This demonstrates that a solver-aware modeling paradigm, which unifies structural mapping with SAT reasoning, can fundamentally enhance LEC efficiency.

### [4] [Vectorizing Quantum Control: A RISC-V Vector Extension Architecture for Scalable Qubit Systems](https://arxiv.org/abs/2607.07372)
*Xiaorang Guo,Kun Qin,Yanbin Chen,Carsten Trinitis,Martin Schulz*

Main category: cs.AR

TL;DR: 本文提出了一种基于RISC-V向量扩展（RVV）的向量化量子控制方法，支持单指令寻址128量子比特、动态旋转参数调整，以及80ns内恢复的硬件暂停-恢复协议，相比基线实现了2.52倍加速。


<details>
  <summary>Details</summary>
Motivation: 现有量子控制处理器依赖定制化指令集，设计复用性差，工具链构建成本高；同时在大规模场景下量子比特寻址和操作调度效率低下。

Method: 基于RISC-V向量扩展（RVV）引擎，设计量子导向的向量化指令集扩展；将参数化旋转信息嵌入指令集；实现基于硬件的暂停-恢复协议以支持电路中间测量。

Conclusion: 向量化量子控制方法有效提升了量子控制处理器的可扩展性和执行效率，同时利用RISC-V生态提高了设计复用性。

Abstract: The Quantum Control Processor (QCP) bridges the gap between compiler toolchains and control electronics, and is responsible for translating compiled quantum circuits into executable instructions that directly manipulate qubits and handle measurement feedback. However, existing designs rely primarily on customized instruction sets, limiting design reuse and requiring significant effort to build supporting toolchains. Furthermore, efficiently addressing qubits and scheduling operations in highly scalable scenarios remains a critical challenge. In this work, we present a vectorized quantum control approach built upon the RISC-V Vector (RVV) engine with a quantum-oriented extension. Leveraging the high parallelism of RVV, our approach can address up to 128 qubits in a single instruction. We also embed parameterized rotation information into the instruction set, enabling dynamic tuning of gate rotations in hybrid quantum-classical programs. To support mid-circuit measurements, we design a hardware-based halt-resume protocol that resumes pipeline execution within 80 $ns$ of receiving the measurement result. Comprehensive evaluation using both RISC-V toolchains and FPGA prototypes demonstrates that our design achieves up to 2.52$\times$ speedup over the baseline in program execution time, with excellent scalability.

### [5] [ATLAS: Automated HLS for DL-Optimized FPGAs](https://arxiv.org/abs/2607.07643)
*Ruthwik Reddy Sunketa,Aman Arora*

Main category: cs.AR

TL;DR: ATLAS是一个全自动流程，能从高层DL模型描述直接生成FPGA硬件实现，利用自定义硬块加速GEMM计算，无需手动RTL设计或显式硬块实例化。


<details>
  <summary>Details</summary>
Motivation: FPGA架构虽集成了领域特定硬块加速DL推理，但使用这些硬块需要手动RTL设计，HLS工具又不原生支持自定义硬块的代码生成，且DL领域需要比C/C++更高层的编程接口（如PyTorch/Keras）。

Method: ATLAS以GEMM为统一抽象层，包含两个组件：(1) hls4ml-GEMM编译器前端，将DL层转换为带架构无关GEMM调用的HLS C代码；(2) GEMM IP Generator架构感知后端，生成基于硬块的RTL包装器，包含分片逻辑、控制FSM和调度元数据。

Result: 在11个DL设计上评估，包括全连接层、卷积层、注意力层以及完整的CNN、MLP和Transformer模型，使用Tensor Slices的FPGA架构，采用Catapult进行HLS、VTR进行实现。

Conclusion: ATLAS实现了从高层DL框架到FPGA硬件的全自动流程，消除了手动RTL设计和显式硬块实例化的需求，提升了生产力和可编程性。

Abstract: FPGA architectures increasingly incorporate domain-specific in-fabric hardblocks to accelerate DL inference, particularly GEMM, which dominates DL computation. To realize the performance gains of these hardblocks, manual RTL design is required: the programmer must understand the hardblock microarchitecture, instantiate them in RTL, and manage tiling and control logic. While programming in C/C++ and using HLS tools has increased the abstraction level and productivity of FPGA engineers, HLS tools do not support code generation for custom hardblocks natively. Prior work has demonstrated that blackbox mechanisms in HLS tools can be used to target custom hardblocks, but this still requires explicit function calls in user-written HLS C and manual creation of RTL IP libraries, significant effort that must be repeated for every layer in a DL model. Furthermore, for DL, an even high-level programming interface, e.g., Pytorch/Keras instead of C/C++, is desirable for improved programmability and user adoption.
  We present ATLAS, a fully automated flow from a high-level DL model description to a hardware implementation on an FPGA with custom in-fabric DL-optimized hardblocks, requiring no manual RTL design or explicit hardblock instantiation from the end user. Our approach uses GEMM as a universal abstraction layer and comprises two components: (1) hls4ml-GEMM, a compiler frontend that transforms DL layers into HLS C code with architecture-agnostic GEMM function calls, and (2) a GEMM IP Generator, an architecture-aware backend that produces hardblock-based RTL wrappers with tiling logic, control FSMs, and scheduling metadata. We evaluate the flow across 11 DL designs, including individual fully connected, convolution, and attention layers, as well as full CNN, MLP, and Transformer models targeting an FPGA architecture with Tensor Slices using Catapult for HLS and VTR for implementation.

</details>
