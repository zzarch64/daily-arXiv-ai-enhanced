### [2] [ORRAM: An OpenROAD-Integrated RAM Generator Using Standard Cells](https://arxiv.org/abs/2607.12244)
*Brayden Louie,Thinh P. Nguyen,Matt Liberty,Austin Rovinski*

Main category: cs.AR

TL;DR: ORRAM是一个基于标准单元的内存生成器，作为OpenROAD的原生模块，无需定制位单元或外部工具即可生成完全布局布线的RAM块，支持多种配置，在SkyWater 130nm上实现了与DFFRAM相当的位密度。


<details>
  <summary>Details</summary>
Motivation: 传统ASIC设计流程中，从RTL推断触发器会产生大量密集互联的实例，严重拖慢设计流程并影响性能。内存编译器虽然是解决方案，但它们是第三方工具，通常依赖于特定PDK，且可能需要基础PDK中没有的专用单元。

Method: ORRAM作为OpenROAD的原生模块，基于标准单元库实现内存生成。它利用OpenROAD的布局布线流程，通过OpenSTA进行时序验证而非SPICE仿真。支持任意字长、字数、掩码粒度、多端口读配置、列复用、锁存器存储和自动PDK无关单元选择。

Result: 在SkyWater 130nm工艺上评估，ORRAM达到了与历史DFFRAM相当的位密度，同时提供了显著扩展的功能集。

Conclusion: ORRAM提供了一种PDK无关、无需外部工具的标准单元内存生成方案，作为开源OpenROAD项目的一部分，可广泛兼容大多数标准单元库。

Abstract: Memory inference remains a significant challenge in turnkey ASIC design flows. Inferring flip-flops from RTL can create thousands of densely interconnected instances which dramatically slow down design flows and impede performance. Memory compilers address this issue, although they are third-party tools which are often PDK-specific and may require specialized cells not in the base PDK.
  To address these shortcomings, we present ORRAM, a standard-cell-based memory generator built as a native module within OpenROAD. Given a standard cell library, ORRAM produces a fully placed and routed RAM block requiring no custom bitcells or external tooling, with timing verification via OpenSTA rather than SPICE simulation. ORRAM supports arbitrary word sizes, word counts, mask granularities, multi-port read configurations, column muxing, latch-based storage, and automatic PDK-agnostic cell selection, making it compatible with most standard cell libraries including sky130hd and NanGate45. Evaluated on SkyWater 130nm, ORRAM matches the bit density of historical DFFRAM results while offering a significantly expanded feature set. The source code is available as part of the OpenROAD project.

### [3] [Emulated Integrity Replica: Enabling Self-Healing on FPGA SoCs via Hierarchical Twins](https://arxiv.org/abs/2607.12298)
*Arsalan Ali Malik,Ali Suvizi,Guru Venkataramani,Aydin Aysu*

Main category: cs.AR

TL;DR: 本文提出EIR（模拟完整性副本），一种用于FPGA SoC的分层数字孪生框架，利用处理系统中的空闲周期来运行粗粒度（Rabbit）和细粒度（Tortoise）双模型，实现自主故障检测与恢复，在降低资源开销的同时达到接近DMR的故障覆盖率。


<details>
  <summary>Details</summary>
Motivation: 传统硬件冗余（DMR/TMR）资源开销大，软件容错方法（ABFT、检查点重启等）延迟/能耗高且覆盖不足，亟需一种在资源受限边缘设备上高效的容错方案。

Method: 提出EIR框架：利用FPGA SoC中PS空闲周期，建立两个数字孪生模型——Rabbit（粗粒度行为模型，快速检测）和Tortoise（细粒度门级模型，精确恢复），定期捕获加速器状态并利用执行速度剖析来平衡性能开销与可靠性。

Result: 在代表性工作负载上，EIR实现了接近DMR的故障覆盖率，同时显著降低了能耗和面积开销。

Conclusion: EIR为严格资源预算下的边缘AI部署提供了一条实用的弹性路径。

Abstract: Convolutional neural networks (CNNs) are increasingly being deployed on system-on-chip (SoC) platforms, where hardware-accelerated inference enables low-latency edge computing. Achieving fault tolerance on these devices remains challenging because conventional redundancy (dual/triple modular redundancy, DMR/TMR) incurs high resource cost, while software-centric methods (e.g., algorithm-based fault tolerance (ABFT), checkpoint-restart, instruction-level duplication, and software watchdogs/assertions) introduce nontrivial latency/energy overheads, reduce model accuracy, or provide inadequate coverage for accelerator-induced faults.
  In this paper, we propose Emulated Integrity Replica (EIR), a hierarchical digital-twin framework for FPGA SoCs that provides autonomous fault detection and recovery. Unlike DMR/TMR, which replicates hardware logic and incurs proportional area and power overheads, EIR avoids fabric-level duplication by exploiting temporal slack in the processing system (PS). During accelerator execution in the programmable logic (PL), the PS typically remains underutilized; EIR capitalizes on these idle cycles to host two complementary twins: (i) Rabbit: a coarse-grained behavioral model for rapid fault detection and (ii) Tortoise: a fine-grained gate-level model that performs precise recovery from checkpointed states. The accelerator state is captured periodically, leveraging the accelerator's execution-speed profiling to balance performance overhead and resilience. Experiments on representative workloads show that EIR achieves high empirical fault coverage relative to a DMR baseline while reducing energy and area under the evaluated fault model and workload assumptions, indicating a practical path to resilient edge-AI deployments under strict resource budgets.

  <summary>Details</summary>

The platform introduces only 1.6 seconds of overhead per simulation, negligible relative to realistic simulation workloads.

Main category: cs.AR

<details>
  <summary>Details</summary>

Method: 硬件方面设计MD-SpMM内核，将稀疏GEMM重组为微密集、Tensor Core对齐的数据流，并采用感知推理的自适应并行；软件方面采用三阶段启发式搜索+约束松弛进行逐层稀疏配置。

To this end, we propose a hardware-software co-design framework for N:M sparse ViT inference. On the hardware side, we design MD-SpMM, an N:M sparse CUDA kernel that reorganizes sparse GEMM into micro-dense, Tensor-Core-aligned dataflow and uses inference-aware adaptive parallelism to sustain utilization. On the software side, we perform layer-wise sparsity search under explicit end-to-end latency budgets using a three-stage heuristic search with constraint relaxation to avoid premature convergence and enable deployment-aware sparsity allocation. Experiments on multiple ViT/Swin models and GPU platforms show that the framework achieves over 2.2x latency speedup while maintaining comparable accuracy and delivering superior accuracy under the same latency constraint. The source code is publicly available at https://github.com/liuganhuo/realizable-nm-sparse-transformer.

### [6] [CLIP-3D: Closed-Loop Evaluation of Performance and Physical Constraints for 3D ICs](https://arxiv.org/abs/2607.12788)
*Shuo Ren,Libo Shen,Yaohui Han,Leilei Jin,Chenghan Wang,Zhen Zhuang,Rongliang Fu,Bei Yu,Tsung-Yi Ho*

Main category: cs.AR

TL;DR: CLIP-3D是一个提前评估3D集成电路布局影响的设计流程，将物理布局的热效应、线延迟和缓存时序反馈到早期架构探索中，以替代传统基于理想时序的架构模拟。


<details>
  <summary>Details</summary>
Motivation: 传统架构模拟器（如gem5）在理想时序下报告IPC，无法反映3D集成中布局决定的热点分布、缓存周期和实际吞吐量，导致早期设计选型忽略了硅上实际会发生的降频效应。

Method: 两阶段流程：第一阶段将架构配置转化为物理块表示（McPAT功耗、CACTI缓存参数、HotSpot兼容的3D堆叠离散化）；第二阶段运行分析型3D热感知布局规划器，利用HotSpot稳态算子的线性性和CMOS功率-频率分解推导闭式持续频率表达式，联合优化跨层宏分配和同层布局，以实际BIPS为目标而非HPWL加温度的加权和。

Result: 提出了一个能够在签核工具介入前，将3D布局驱动的热、线延迟和缓存效应暴露给早期架构探索的shift-left流程。

Conclusion: CLIP-3D通过将物理效应提前到架构探索阶段，使得3D-IC设计能在更早期选择出实际硅上性能更高的设计方案，避免选中后续会因热或线延迟而降频的设计。

Abstract: 3D integration packs more power into a smaller footprint, so a candidate design's actual throughput depends on its layout: which macro sits on which tier, where the hot spot lands, and how cache geometry maps to access cycles. Architectural simulators like gem5 report IPC under idealized timing. They do not produce the per-block power map, the cache cycle counts, or the 3D layout that decide the realized billion-instructions-per-second (BIPS), so early-stage 3D-IC exploration selects designs without accounting for the effects that decide whether they throttle on silicon. We present CLIP-3D, a shift-left flow that exposes 3D layout-driven thermal, wire, and cache effects to early-stage architectural exploration before any sign-off tool is invoked. The first stage lifts an architectural configuration into a physical block representation: McPAT for per-block dynamic and leakage power, CACTI for cache geometry and access cycles, and a HotSpot-compatible 3D stack discretization. The second stage runs an analytical 3D thermal-aware floorplanner over that representation. The floorplanner objective embeds a closed-form sustained-frequency expression derived from the linearity of HotSpot's steady-state operator and the standard CMOS power-frequency decomposition. Cross-tier macro assignment and in-plane placement are co-optimized for the realized BIPS rather than for a half-perimeter wirelength (HPWL)-plus-temperature surrogate with hand-tuned weights.

</details>
