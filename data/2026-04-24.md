### [1] [Ternary Memristive Logic: Hardware for Reasoning Realized via Domain Algebra](https://arxiv.org/abs/2604.20891)
*Chao Li*

Main category: cs.AR

TL;DR: 本文提出一种基于忆阻交叉阵列的硬件推理架构，将每个存储单元直接编码领域内的三值逻辑断言（真/假/未定义），通过结构保持映射将领域代数转化为物理电路拓扑，实现了无需符号解释的、读取即回答的硬件推理。在ICD-11呼吸疾病分类芯片设计上验证了其可行性。


<details>
  <summary>Details</summary>
Motivation: 传统忆阻交叉阵列中每个单元仅存储权重数值，需要聚合与解码才能获得语义信息。本文希望从根本改变这一范式，让每个结直接存储一个完整的逻辑断言，使硬件布局本身即体现推理语义。

Method: 将领域代数结构保持地映射到交叉阵列拓扑：领域对应隔离阵列、特化对应定向布线、关系类型控制继承门、跨域链接用显式寄存器实现。利用三态电阻状态（真/假/未定义）编码逻辑值。设计ICD-11呼吸疾病分类芯片（1,247实体，约136k 1T1R结）实现域限定、三值逻辑、传递级联、类型继承和跨轴查询。

Result: 行为仿真（sigma_log=0.15，SNR=20dB）显示每项任务10万次试验中零错误运行，具有宽容限裕度。

Conclusion: 先前工作在软件层面统一表示与计算，本文在硬件层面实现统一：读取一个结即可回答一个问题，无需符号解释。

Abstract: Memristive crossbars store numerical weights needing aggregation and decoding; a single junction means nothing alone. This paper presents a fundamentally different use: each junction stores a complete, domain-scoped logical assertion (holds/negated/undefined). Ternary resistance states encode these values directly. We establish a structure-preserving mapping from a domain algebra to crossbar topology: domains become isolated arrays, specialization becomes directed wiring, relation typing controls inheritance gates, and cross-domain links become explicit registers. The physical layout thus embodies the algebra; changing wiring changes reasoning semantics. We detail an ICD-11 respiratory disease classification chip (1,247 entities, ~136k 1T1R junctions) enabling domain scoping, three-valued logic, transitive cascade, typed inheritance, and cross-axis queries. Behavioral simulation (sigma_log=0.15, SNR=20dB) shows error-free operation across 100,000 trials per task with wide tolerance margins. Where prior work unified representation and computation in software, this work unifies them in hardware: reading one junction answers one question, without symbolic interpretation.

### [2] [AnalogMaster: Large Language Model-based Automated Analog IC Design Framework from Image to Layout](https://arxiv.org/abs/2604.20916)
*Xian Rong Qin,Yong Zhang,Ying Hu,Tao Su,Bo-Wen Jia,Ning Xu*

Main category: cs.AR

TL;DR: AnalogMaster 是一个基于大语言模型的端到端模拟集成电路设计自动化框架，从电路图像到网表生成、参数优化、布局布线实现全流程自动化，在15个代表电路上GPT-5的Pass@1和Pass@5分别达到92.9%和99.9%。


<details>
  <summary>Details</summary>
Motivation: 现有模拟IC设计自动化工具仅关注个别阶段（如器件尺寸、布局或布线），仍需大量人工干预；数字IC设计的LLM自动化方法无法直接迁移到模拟IC领域，因为存在性能指标强耦合、非结构化电路原理图图像、以及现有方法仅处理孤立阶段等挑战。

Method: 提出AnalogMaster框架，包含：(1) 联合推理机制，结合上下文学习和意图推理实现图像到网表的准确转换；(2) 参数搜索智能体，集成自增强提示工程和上下文截断，用于器件尺寸优化和后续物理设计；(3) 统一流水线涵盖图像到网表生成、参数优化、布局和布线。

Result: 在15个复杂度不同的代表电路上验证，多个模型表现一致；GPT-5在Pass@1和Pass@5上分别达到92.9%和99.9%的成功率。

Conclusion: AnalogMaster验证了所提框架的有效性和鲁棒性，为将LLM应用于全栈模拟IC设计自动化建立了实用范式。

Abstract: Design automation has the potential to substantially improve the efficiency of analog integrated circuit (IC) design. However, existing algorithms and tools typically focus on individual stages, such as device sizing, placement, or routing, and still require significant manual intervention to complete the full design flow. While large language models (LLMs) have recently demonstrated remarkable success in automating digital IC design workflows, these advances cannot be directly transferred to analog IC design. Key challenges include strongly coupled performance metrics, the predominance of unstructured circuit schematic images, and the fact that most prior approaches address only isolated stages of the analog design process, limiting their ability to capture end-to-end performance impact. To address these challenges, we propose AnalogMaster, an extensible, LLM-based framework that enables end-to-end automation of analog IC design through a unified pipeline spanning circuit image-to-netlist generation, parameter optimization, placement, and routing. AnalogMaster integrates a joint reasoning mechanism that leverages in-context learning and intent reasoning to achieve accurate and robust image-to-netlist conversion. A parameter search agent integrating self-enhanced prompt engineering and context truncation is developed for effective device sizing and downstream physical design. Experimental evaluations on 15 representative circuits with varying levels of complexity demonstrate strong and consistent performance across multiple models. In particular, GPT-5 achieves success rates of 92.9% and 99.9% on Pass@1 and Pass@5, respectively. These results validate the effectiveness and robustness of the proposed framework and establish a practical paradigm for applying LLMs to full-stack analog IC design automation.

### [3] [Efficient Batch Search Algorithm for B+ Tree Index Structures with Level-Wise Traversal on FPGAs](https://arxiv.org/abs/2604.21117)
*Max Tzschoppe,Martin Wilhelm,Sven Groppe,Thilo Pionteck*

Main category: cs.AR

TL;DR: 本文提出了一种基于FPGA的B+树搜索算法，通过批量处理搜索键、按层级遍历节点来减少全局内存访问并提高节点复用率。在AMD Alveo U250上实现，相比单线程CPU实现获得4.9倍加速，相比16线程CPU实现获得2.1倍加速。


<details>
  <summary>Details</summary>
Motivation: 传统B+树搜索算法在CPU上执行时存在大量内存访问开销，而FPGA具有并行处理和低延迟的优势，探索FPGA加速B+树搜索以提升性能。

Method: 基于B+树的搜索算法，在FPGA上按层级批量处理搜索键，实现节点遍历与复用，利用高综合(HLS)方法设计灵活可配置的搜索内核，支持可调批量大小、节点大小和树深度。

Result: 在AMD Alveo U250上，批量1000个键、100万条B+树条目、阶数为16时，单内核FPGA设计相比单线程CPU实现加速4.9倍；四个并行内核相比16线程CPU实现加速2.1倍。

Conclusion: 基于FPGA的批量B+树搜索算法能有效减少内存访问、提高节点复用，实现显著的加速效果，且通过HLS方法保证了设计的灵活性与可配置性。

Abstract: This paper introduces a search algorithm for index structures based on a B+ tree, specifically optimized for execution on a field-programmable gate array (FPGA). Our implementation efficiently traverses and reuses tree nodes by processing a batch of search keys level by level. This approach reduces costly global memory accesses, improves reuse of loaded B+ tree nodes, and enables parallel search key comparisons directly on the FPGA. Using a high-level synthesis (HLS) approach, we developed a highly flexible and configurable search kernel design supporting variable batch sizes, customizable node sizes, and arbitrary tree depths. The final design was implemented on an AMD Alveo U250 Data Center Accelerator Card, and was evaluated against the B+ tree search algorithm from the TLX library running on an AMD EPYC 7542 processor (2.9 GHz). With a batch size of 1000 search keys, a B+ tree containing one million entries, and a tree order of 16, we measured a 4.9x speedup for the single-kernel FPGA design compared to a single-threaded CPU implementation. Running four kernel instances in parallel on the FPGA resulted in a 2.1$\times$ performance improvement over a CPU implementation using 16 threads.

### [4] [Enabling Mixed criticality applications for the Versal AI-Engines](https://arxiv.org/abs/2604.21124)
*Vincent Sprave,Martin Wilhelm,Daniele Passaretti,Alberto Garcia-Ortiz,Thilo Pionteck*

Main category: cs.AR

TL;DR: 本文提出了一种动态任务调度基础设施，首次在AMD Versal AIE上实现混合关键性系统（MCS）任务切换，克服了静态数据流图映射的限制，通过动态分配不同关键性级别的任务到AIE tile池，在自动驾驶案例中实现了空闲时间减少65.5%、执行时间开销低于0.002%、低关键性任务吞吐量翻倍的效果。


<details>
  <summary>Details</summary>
Motivation: AMD Versal AIE虽具有强大的并行处理能力，但现有数据流图与任务的静态映射方式无法在运行时动态分配不同关键性级别的任务，限制了其在混合关键性系统（如自动驾驶、航空、医疗）中的应用。本文旨在填补这一空白，首次探索AIE在MCS场景下的潜力。

Method: 提出动态任务调度基础设施，支持AIE上运行时任务切换；基于该设施设计MCS架构，根据系统关键性模式将不同关键性任务动态分配到AIE tile池；对任务调度器的控制逻辑、上下文切换和数据拷贝操作进行时序分析。

Result: 所提方案在自动驾驶工作负载下，AIE利用率最大化，空闲时间减少65.5%，执行时间开销低于0.002%，低关键性任务吞吐量翻倍。

Conclusion: 动态任务调度基础设施开销低、方差小，适用于混合关键性系统；首次实现了AIE在MCS中的并行处理能力利用，克服了静态映射局限。

Abstract: Adaptive Systems-on-Chips (SoCs) are increasingly being used in mixed criticality systems (MCSs), such as in autonomous driving, aviation and medical systems. In this context, AMD has proposed the Versal SoC, which has a heterogeneous architecture including, among other components, an Artificial Intelligence Engine (AIE), which is a 2D array of processors and memory tiles designed for AI and signal processing workloads. While this AIE offers significant potential for accelerating real-time data processing tasks, this has not yet been explored in the context of MCSs since individual tasks with different criticality levels cannot be dynamically assigned to tiles due to the static mapping of dataflow graphs and tasks. In this work, we propose a dynamic task dispatching infrastructure that enables task switching on the AIE at runtime. Based on this infrastructure, we present an MCS design that dynamically assigns tasks of different criticality to a pool of AIE tiles, depending on the criticality mode of the system. Our approach overcomes the limitations of static dataflow graph mappings and, for the first time, exploits the parallel processing capabilities of the AIE for MCSs. We also present a comprehensive timing analysis of the overhead introduced by the task dispatcher infrastructure, focusing on control logic, context switching and data copy operations. This shows that these operations have low variance and are negligible compared to the overall execution time, demonstrating that our infrastructure is suitable for MCSs. Finally, we evaluate the proposed infrastructure using an autonomous driving workload with tasks that have variable execution times and different criticality levels. In this case study, we maximized AIE utilization, reducing idle time by 65.5 %, while measuring an execution time overhead of less than 0.002 %, and doubling the throughput of low-criticality tasks.
